# 撰写一篇 AI 学习知识点文档

按以下步骤在本知识库中新增一篇高质量、符合规范的知识点文档。

## 步骤 1：确定分类与序号

1. 打开 `docs/00-元规范/000-分类总览与知识图谱.md`，确认该知识点应归入哪个「NN-分类名」目录；若无合适分类，先与用户确认是否新建分类目录（新建时同步更新总览表与依赖图谱）。
2. 查看目标分类目录下已有文件的三位序号，为新文档选择下一个可用序号（`000` 永远留给分类总览文档）。
3. 文件路径形如 `docs/NN-分类名/NNN-知识点名.md`。

## 步骤 2：核实知识点（保证正确性）

- 对不确定的概念、公式、数据，使用 `deepwiki` 或 `context7` MCP 查询权威来源核实。
- 优先采用教科书/论文/官方文档中的共识表述；存在争议处显式标注。

## 步骤 3：按标准结构撰写

严格采用以下结构（可按需增减，但"通俗解读"和"案例"必须保留）：

```markdown
# NNN · 知识点标题

> 一句话点明本文要解决的核心问题。

## 一、直觉与通俗解读   # 大白话/类比讲清"是什么、为什么"
## 二、严谨定义与原理   # 教授级：定义、公式（$...$/$$...$$）、推导、前提
## 三、案例解析         # 可复现案例；代码放 code/ 镜像目录并相对链接引用
## 四、常见误区与边界   # 易错点、适用范围、反例
## 五、小结与延伸阅读   # 要点回顾 + 关联文档相对链接
```

写作要点：
- 先直觉后严谨；通俗类比之后必须回到准确定义。
- 公式用 LaTeX；对比用表格；流程/关系用 Mermaid。
- 案例尽量经典、可复现，必要时在 `code/NN-分类名/NNN-知识点/` 下补充可运行代码。

## 步骤 4：更新分类总览

- 若新增了新知识点，回到本分类的 `000-分类总览与知识图谱.md`，把新知识点补进知识图谱与学习建议中（保持图谱反映真实关系）。

## 步骤 5：校验与预览

1. 运行 `npm run docs:validate` 校验命名与结构。
2. 运行 `npm run dev` 本地预览，确认公式、Mermaid 图谱、内部链接均正常渲染。
3. 确认 `npm run build` 不因死链失败。
