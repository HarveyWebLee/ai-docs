# 撰写一篇 AI 学习知识点文档

按以下步骤在本知识库中新增一篇高质量、符合规范的知识点文档。

## 步骤 1：确定分类与序号

1. 打开 `docs/00-元规范/000-分类总览与知识图谱.md`，确认该知识点应归入哪个「NN-分类名」目录；若无合适分类，先与用户确认是否新建分类目录（新建时同步更新总览表与依赖图谱）。
2. 查看目标分类目录下已有文件的三位序号，为新文档选择下一个可用序号（`000` 永远留给分类总览文档）。
3. 文件路径形如 `docs/NN-分类名/NNN-知识点名.md`。

## 步骤 2：核实知识点（保证正确性）

- 对不确定的概念、公式、数据，使用 `deepwiki` 或 `context7` MCP 查询权威来源核实。
- 优先采用教科书/论文/官方文档中的共识表述；存在争议处显式标注。

## 步骤 3：按"通俗优先"结构撰写

本仓库面向零基础读者，**顺序原则不可违背：先通俗 → 后专业 → 再对齐**。推荐结构（"一句话/打比方/解决什么问题/与大白话对齐的专业视角/案例"必须保留）：

```markdown
# NNN · 知识点标题

> 本文用大白话回答：……（点明核心疑问）

## 一、一句话先说清         # 一句"人话"讲清"是什么"
## 二、打个比方             # 生活化类比，建立画面
## 三、它到底解决什么问题   # 分「问题1/问题2/…」，配具体例子或对比表格
## 四、专业视角（与大白话对齐）  # 定义/公式（$...$/$$...$$），每处回扣前面的大白话
## 五、案例解析             # 可复现案例；代码放 code/ 镜像目录并相对链接引用
## 六、常见误区与边界       # 易错点、适用范围、反例
## 七、一句话总结           # 用最朴素的话收尾
```

通俗优先三步法（写每个概念都照做）：
1. **先大白话**：一句人话说清本质；专有名词首次出现立刻用括号解释。
2. **打比方 + 讲"解决什么问题"**：生活化类比 + 拆成几个问题，每个配例子或"朴素做法 vs 更好做法"对比表格。
3. **专业对齐**：再给定义/公式（LaTeX），并立刻回扣前面的大白话（"这个公式就是前面那句……"）。

其它：公式用 LaTeX；对比用表格；流程/关系用 Mermaid；案例经典、可复现，必要时在 `code/NN-分类名/NNN-知识点/` 补可运行代码。**对照黄金样板 `docs/03-深度学习基础/001-神经网络结构.md` 的"为什么必须有非线性激活函数"一节。**

## 步骤 4：更新分类总览

- 若新增了新知识点，回到本分类的 `000-分类总览与知识图谱.md`，把新知识点补进知识图谱与学习建议中（保持图谱反映真实关系）。

## 步骤 5：校验与预览

1. 运行 `npm run docs:validate` 校验命名与结构。
2. 运行 `npm run dev` 本地预览，确认公式、Mermaid 图谱、内部链接均正常渲染。
3. 确认 `npm run build` 不因死链失败。
