# 000 · 分类总览与知识图谱

> 本页是「AI 工程与 MLOps」分类的导读，串联本分类知识点并绘制知识图谱。本分类关注"如何把模型可靠地送进生产并持续运营"。

## 一、本分类学什么

模型训练成功只是起点，让它在真实业务中**稳定、可复现、可持续地创造价值**才是工程挑战。本分类覆盖 ML 系统的全生命周期：

- 喂给模型的数据从哪来、怎么处理——[001 · 数据管线与特征工程](./001-数据管线与特征工程.md)
- 如何让实验可追踪、可复现——[002 · 实验管理与可复现性](./002-实验管理与可复现性.md)
- 如何把模型变成可调用的服务——[003 · 模型部署与推理服务](./003-模型部署与推理服务.md)
- 上线后如何发现"模型悄悄变差"——[004 · 监控与模型漂移](./004-监控与模型漂移.md)

## 二、通俗理解本分类

如果说训练模型像"**研发一道菜**"，那么 MLOps 就是"**开一家能长期稳定出餐的餐厅**"：

- **数据管线**是稳定的食材供应与预处理；
- **实验管理**是把每次配方都记录在案，随时能复刻；
- **部署服务**是把菜做成标准化流程、能高效上桌；
- **监控**是持续尝味道——一旦食材变了、口味变了（数据漂移），立刻预警并调整。

## 三、知识图谱


这是一个**闭环**：监控发现问题 → 触发再训练 → 重新部署，持续迭代。

## 四、学习建议

1. 牢记"**数据 > 模型**"：真实项目里数据质量与管线往往比模型算法更决定成败。
2. 把"可复现"当成第一原则：固定随机种子、记录版本与依赖。
3. 部署后监控不可省——模型会随世界变化而"变质"。
4. 本分类是把 [02-机器学习基础](../02-机器学习基础/000-分类总览与知识图谱.md) 与 [03-深度学习基础](../03-深度学习基础/000-分类总览与知识图谱.md) 落地的关键。

## 五、小结

- MLOps 管理 ML 系统全生命周期：数据 → 训练 → 部署 → 监控 → 再训练的闭环。
- 核心原则：数据质量优先、实验可复现、上线可监控。
- 工程能力决定了模型能否真正产生业务价值。
