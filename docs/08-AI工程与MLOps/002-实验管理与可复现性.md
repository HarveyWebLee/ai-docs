# 002 · 实验管理与可复现性

> 本文用大白话回答：为什么机器学习实验特别难"照着再跑一遍就出一样的结果"？想复刻一次结果得记下哪些东西？版本管理和实验跟踪工具到底帮我们省了什么事？
>
> 读完你会知道：一次好结果如果没法复现，就等于不可信；把每次实验的"完整配方"记全，才能随时一键复刻、稳步改进。

## 一、一句话先说清

**可复现性 = 把每次实验的"全部配方"完整记下来，任何时候都能照着一键复刻出同样的结果。**

做机器学习就像做科学实验——**如果一次好结果无法被复现，它就不可信、也无法改进**。而 ML 实验又特别容易"复现失败"，因为影响结果的因素太多：代码、数据、超参数、随机种子、依赖库版本、硬件……改动其中任何一个，结果都可能对不上。

## 二、打个比方：复刻一道菜

你在饭店吃到一道特别好吃的菜，回家想复刻，结果做出来味道完全不一样。为什么？因为"照着做"其实远不止抄一份菜谱：

| 复刻一道菜要对齐的 | 复刻一次实验要对齐的 | 不对齐会怎样 |
| --- | --- | --- |
| 菜谱步骤 | 代码 | 步骤不同，做法就不同 |
| 食材批次/产地 | 数据（版本） | 换了批食材，味道就变 |
| 盐糖火候的用量 | 超参数、随机种子 | 调料量一变，成品就变 |
| 灶具、锅、燃气 | 依赖库版本、硬件 | 换个灶，火力不一样 |

关键直觉：**只抄菜谱（只留代码）远远不够**，食材、用量、灶具都得对上，才能做出同一个味道。这就是实验管理要解决的问题。

## 三、它到底解决什么问题

### 问题 1：想复刻结果，到底要记下哪些东西？——追踪"四件套"

要复现一次实验，至少需固定并记录下面这"四件套"：

| 维度 | 内容 |
| --- | --- |
| **代码** | Git commit 哈希（代码版本） |
| **数据** | 数据集版本/快照（如 DVC 管理） |
| **配置** | 超参数、特征列表、随机种子 |
| **环境** | 依赖库版本、镜像（如 Docker）、硬件信息 |

只要这四件套固定，结果就应可复现。（对照上面的比方：代码=菜谱、数据=食材、配置=用量、环境=灶具。）

### 问题 2：为什么"随机"会让结果对不上？——控制随机性

ML 里到处是"随机"：权重初始化、数据打乱、Dropout、数据增强等，都会用到随机数。今天和明天各跑一次，起点就不一样，结果自然对不上。

- **朴素做法**：不管随机，跑一次是一次 → 每次结果都飘。
- **更好做法**：**固定随机种子（seed）**，让"随机"每次都从同一个起点出发，可重复。

此外还要注意：某些 GPU 算子存在**非确定性**（同样输入也可能有微小差异），必要时开启确定性模式（可能牺牲速度）。

### 问题 3：几十上百次实验，怎么知道"哪次改动带来了提升"？——实验跟踪与模型注册表

**实验跟踪**：用实验跟踪工具（如 MLflow、Weights & Biases）自动记录每次运行的：

- 参数（超参、配置）
- 指标（loss、准确率随步数变化）
- 产物（模型权重、图表）

并支持多次实验横向对比，找出"哪个改动带来了提升"——相当于给每次实验建一本"实验台账"，好坏一目了然。

**模型注册表（Model Registry）**：训练好的模型登记到模型注册表，带版本、阶段（staging/production）、来源实验链接，实现从"实验"到"上线模型"的可追溯治理——相当于给每个"成品菜"贴上标签：哪批做的、现在是试菜还是正式上桌、出自哪次实验。

## 四、专业视角（与大白话对齐）

把前面的通俗说法收敛成严谨定位：

- **可复现性**：固定并记录"代码 + 数据 + 配置 + 环境"这四个维度后，同一实验应能得到一致结果。
  - 对齐：这就是"把整份配方记全，才能复刻同一个味道"。
- **随机种子（seed）**：让所有随机来源从同一初始状态出发，从而结果可重复。
  - 对齐：即前面说的"让随机每次都从同一个起点出发"。
- **实验跟踪 / 模型注册表**：前者记录并对比每次运行的参数、指标、产物；后者对模型做版本与阶段治理。
  - 对齐：分别是"实验台账"与"成品菜的标签与去向登记"。

## 五、案例解析：为什么"同样的代码"结果不一样

同事说他的模型准确率 92%，你拉了同一份代码却只跑出 88%，排查发现：

1. 他用的是 `数据集 v2`（清洗过），你用的是 `v1` → **数据版本不同**。
2. 他固定了 `seed=42`，你没设种子 → **随机初始化不同**。
3. 他的 `numpy/torch` 版本与你不同 → **依赖版本不同**。

把三者对齐（同数据版本、同种子、同依赖）后，你也复现出了 92%。**这说明"代码相同"远不足以复现结果**——数据、种子、环境同样是实验的一部分，必须一并记录与固定。这正是实验管理的价值所在。

## 六、常见误区与边界

- **误区："记录代码就够了"**：数据、种子、环境同样关键，缺一不可。
- **误区："复现只为存档"**：可复现是可靠调优、协作与合规审计的基础。
- **边界**：分布式训练、混合精度、GPU 非确定性可能导致微小差异，实践中追求"结果在合理误差内可复现"。

## 七、一句话总结

- 可复现需固定并记录"代码 + 数据 + 配置 + 环境"四件套，并控制随机种子。
- 实验跟踪工具记录参数/指标/产物并支持对比；模型注册表实现模型治理。
