# 007 · 视频生成基础

> 本文用大白话回答：文生图已经挺熟了，文生视频难在哪？为什么视频容易「闪烁、脸变来变去」？SVD、Sora 这类模型在解决什么问题？
>
> 读完你会知道：视频生成 = **在时间轴上再叠一维「前后帧要连贯」**——算力暴涨、一致性是头号难题。

## 一、一句话先说清

**视频生成 = 在图像生成基础上，建模**帧序列** $\{x_1,x_2,\dots,x_T\}$，使相邻帧运动合理、主体一致、符合文本描述。**

常见路线：

- **图像模型 + 时间模块**：在 U-Net/Transformer 上加 temporal attention / 3D 卷积（如 Stable Video Diffusion）；
- **原生视频扩散**：直接在视频 latent 上扩散（如部分 Sora、CogVideo 类路线）；
- **两阶段**：先生成关键帧，再插帧（image-to-video 常见）。

## 二、打个比方：连环画 vs 单张插画

| 单张文生图（见 [001 扩散](./001-扩散模型Diffusion基础.md)） | 文生视频 |
| --- | --- |
| 画**一张**海报 | 画**一本连环画**，还要动作连贯 |
| 只关心「这一帧好不好看」 | 还要「上一帧和下一帧像同一场戏」 |
| 算一次 512×512 | 等价于算 $T$ 帧，显存与时间 ×$T$ 量级 |

核心难点：**时间一致性（temporal consistency）**——人物别这张脸、下一张换一张；背景别闪烁；运动别违反物理直觉。

## 三、它到底解决什么问题

### 问题 1：为什么比文生图难一个数量级

数据是 5D 张量：$(B, T, C, H, W)$（批、时间、通道、高、宽）。同样分辨率下，**计算与显存随帧数 $T$ 线性甚至超线性增长**。

| 因素 | 文生图 | 文生视频 |
| --- | --- | --- |
| 输出维度 | 1 帧 | $T$ 帧（常 14~121） |
| 一致性约束 | 无 | 跨帧外观、运动 |
| 训练数据 | 图文对海量 | 高质量视频+文本更稀缺 |

工程上见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)：视频推理是成本大户。

### 问题 2：时间一致性怎么保证

**朴素做法**：每帧独立用 SD 生成——人物、光照、背景帧帧不同，**闪烁严重**。

**更好做法**（常见技术组合）：

| 技术 | 大白话 |
| --- | --- |
| **Temporal Attention** | 让第 $t$ 帧「看见」前后帧，学一致运动 |
| **3D 卷积 / 时空 U-Net** | 同时在空间与时间滑窗 |
| **Motion / 光流条件** | 喂入运动向量，指定怎么动 |
| **Image-to-Video（I2V）** | 第一帧锁死，只生成后面怎么动（SVD 常见） |
| **噪声/init 共享** | 相邻帧 latent 相关，减少随机跳变 |

> 对齐：Temporal Attention 就是「画下一格前先瞄一眼上一格」，避免连环画人物换脸。

### 问题 3：主流产品形态

- **Stable Video Diffusion（SVD）**：输入**一张图** + 可选 motion bucket，输出短视频；适合「让照片动起来」；
- **Text-to-Video（T2V）**：纯文本生成整段（CogVideoX、Kling、Runway Gen-3、OpenAI Sora 等）；
- **Video-to-Video**：原视频 + prompt 改风格/改内容；
- 与 [004 ControlNet](./004-ControlNet与可控生成.md)、[005 LoRA](./005-LoRA低秩微调.md) 扩展：姿态轨迹、角色 LoRA 用于视频仍属前沿/插件生态。

### 问题 4：评估看什么

- **视觉质量**：单帧清晰度、伪影；
- **时间一致性**：主体 ID、背景稳定（可用 CLIP 帧间相似度等代理）；
- **文本对齐**：是否做了 prompt 里的动作；
- **运动合理性**：是否抖动、是否违反物理（仍常靠人评）。

## 四、专业视角（与大白话对齐）

### 4.1 Image-to-Video 流程（SVD 类）

1. 输入图像经 VAE 编码为 $z_0$；
2. 在 latent 序列上扩散/flow，长度 $T$，条件含首帧、fps、motion 强度；
3. 时空 U-Net 去噪，含 **spatial + temporal** 层；
4. VAE 解码为 $T$ 帧，再合成 mp4/gif。

训练常：大规模短视频 +  caption；有时只用视频无文本则自监督或伪 caption。

### 4.2 与 Flow Matching / 扩散的关系

视频模型多在 [001 扩散](./001-扩散模型Diffusion基础.md) 或 [006 Flow Matching](./006-Flow-Matching基础.md) 框架上，把 $x$ 扩展为 $x_{1:T}$，损失在整段 latent 上计算。采样步数 × 帧数 → **延迟极高**，需蒸馏、少步采样、降分辨率生成再超分。

### 4.3 工程参数直觉

| 参数 | 影响 |
| --- | --- |
| 帧数 $T$ | 越长越连贯叙事可能越好，成本越高 |
| FPS | 运动流畅度 |
| Motion bucket | I2V 运动幅度（SVD） |
| 分辨率 | 512² 常见；1080p 多靠后处理 |

## 五、案例解析：I2V「让照片动起来」

输入：一张静态「猫坐在窗台上」照片 + prompt `"cat looking outside, gentle breeze"`。

1. **错误做法**：用文生图连抽 24 张独立图 → 猫品种、窗台细节帧帧变，像幻灯片 glitch。
2. **I2V 做法**：首帧锁定输入图，模型只预测后续 14~25 帧 latent 的去噪轨迹，temporal 层约束同一只猫、同一窗口。
3. 输出：2~4 秒 mp4，头微动、窗外云动；若 motion 过大仍可能崩坏（爪变形）——当前模型边界。

## 六、常见误区与边界

- **误区："视频模型 = 图模型多跑几次"**：缺少时间建模必然闪烁；需专门架构与训练。
- **误区："Sora 级别人人可本地跑"**：长视频、高一致、复杂物理仍依赖超大算力与数据；消费级多为短 clip、低分辨率。
- **边界**：音频、口型同步（唇形）需另接 TTS + 对口模型；长分钟级叙事仍在研究。

## 七、一句话总结

- 视频生成 = 图像生成 + 时间维；核心是跨帧一致性与运动合理性。
- I2V（如 SVD）锁首帧较稳；T2V 更难、成本更高；与扩散/Flow、ControlNet、LoRA 可组合扩展。
