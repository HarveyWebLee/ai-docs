# 005 · LoRA 低秩微调

> 本文用大白话回答：LoRA 为什么只训"一小撮参数"就能定制 Stable Diffusion 画风、或大模型说话风格？低秩 $BA$ 到底在改什么？和全量微调比省在哪？
>
> 读完你会知道：LoRA = **冻住原模型，只学一块「低秩补丁」**——像给大画贴小贴纸，换任务只换贴纸，底图不用重画。

## 一、一句话先说清

**LoRA（Low-Rank Adaptation，低秩适配）= 冻结预训练权重 $W_0$，只额外训练低秩矩阵 $B,A$，使有效权重变为 $W = W_0 + BA$，用极少参数量实现任务/custom 风格适配。**

在生成式 AI 里常见用途：

- **LLM**：对话、领域问答（详见 [05/003 预训练与微调](../05-大语言模型与Transformer/003-预训练与微调.md)）；
- **Stable Diffusion**：固定角色脸、特定画风、概念物体（"LoRA 模型"下载即用）；
- 与 [004 ControlNet](./004-ControlNet与可控生成.md) 组合：结构用 ControlNet，风格/角色用 LoRA。

## 二、打个比方：底图 + 可替换贴纸

想象一张已经画好的**通用底图**（预训练 $W_0$）：

| 做法 | 类比 | 代价 |
| --- | --- | --- |
| **全参数微调** | 把整张底图重画一遍 | 每个任务存一整张新画（GB 级） |
| **LoRA** | 底图不动，只贴一层**小贴纸** $BA$ 改风格 | 每个任务只存贴纸（几 MB~几十 MB） |

换任务 = 换贴纸，底图共享——这就是 Civitai 上大量 SD LoRA 即插即用的原因。

## 三、它到底解决什么问题

### 问题 1：全量微调大模型，显存和存储都扛不住

7B LLM 或 SD U-Net 全参数微调，需要更新数十亿权重，且**每个下游任务一份完整 checkpoint**。

LoRA 只训 $A\in\mathbb{R}^{r\times k}$、$B\in\mathbb{R}^{d\times r}$，秩 $r$ 通常 4~128：

$$
W = W_0 + \frac{\alpha}{r} BA
$$

$\alpha$ 为缩放超参。参数量从 $d\times k$ 降到 $r(d+k)$，$r\ll \min(d,k)$ 时节省几个数量级。

> 对齐：$BA$ 就是"贴纸"——$B$ 把 $r$ 维小空间映射回输出维，$A$ 把输入压到 $r$ 维；秩 $r$ 小 = 贴纸薄，改动范围有限但常够用。

### 问题 2：为什么「低秩」往往够用

微调带来的权重变化 $\Delta W$ 在 many 任务上**近似低秩**——有效自由度远小于矩阵全尺寸。LoRA 显式用低秩结构去拟合 $\Delta W$，而非记忆整个 $W$。

| 维度 | 全参数微调 | LoRA（$r=16$） |
| --- | --- | --- |
| 可训练参数 | $d \times k$（百万~十亿） | $16(d+k)$（通常 <1%） |
| 每任务存储 | 完整模型 | 仅 $A,B$ 小文件 |
| 推理 | 合并或旁路 | 可合并进 $W$ 无额外延迟 |

### 问题 3：生成式场景里 LoRA 训什么

**Stable Diffusion LoRA** 通常注入 U-Net 的 Cross-Attention 层（$W_Q,W_K,W_V,W_O$）或 FFN，用 10~100 张风格/角色图 + caption 训练数小时即可。

工作流：基座 SD + 加载 LoRA 权重（weight 0~1 可调）+ prompt → 定制输出。

## 四、专业视角（与大白话对齐）

### 4.1 前向计算

原层：$\mathbf{h} = W_0 \mathbf{x}$。

加 LoRA 后：

$$
\mathbf{h} = W_0 \mathbf{x} + \frac{\alpha}{r} B A \mathbf{x}
$$

训练时 $W_0$ 梯度为 0（冻结）；只更新 $A,B$。推理时可预先合并 $W' = W_0 + \frac{\alpha}{r}BA$，与原始层一次矩阵乘。

### 4.2 与其它 PEFT 的关系

| 方法 | 思想 | 典型场景 |
| --- | --- | --- |
| LoRA | 低秩增量 $BA$ | LLM、SD 最主流 |
| Prefix / Prompt Tuning | 只训软 prompt 向量 | 轻量 NLP |
| Adapter | 层间插小模块 | 多任务切换 |
| ControlNet | 旁路结构条件 | 结构控制（见 [004](./004-ControlNet与可控生成.md)） |

LoRA 与 ControlNet **可叠加**：ControlNet 管姿势，LoRA 管画风。

### 4.3 实践要点

- **秩 $r$**：越大表达能力越强、越易过拟合；SD 常用 4~32，LLM 常用 8~64；
- **目标层**：Attention 层往往收益最大；
- **学习率**：LoRA 层可用较大 lr（$10^{-4}$~$10^{-3}$），$W_0$ 不动；
- **多 LoRA 合并**：注意权重冲突，可依次加载或训练融合。

## 五、案例解析：参数量对比

设某 Attention 投影 $W_0 \in \mathbb{R}^{768\times 768}$（约 59 万参数）。

| 方案 | 可训练参数量 |
| --- | --- |
| 全微调该层 | 589,824 |
| LoRA $r=8$ | $8\times(768+768)=12{,}288$（约 **2.1%**） |
| LoRA $r=16$ | 24,576（约 **4.2%**） |

10 层类似注入，总 LoRA 参数仍远小于全模型。Civitai 上一个 SD LoRA 文件常见 10~200 MB，而完整 SD 约 2~7 GB。

## 六、常见误区与边界

- **误区："LoRA 完全等于免费午餐"**：极复杂迁移、需大幅改行为时，全参或更大 $r$ 可能仍更好。
- **误区："LoRA 只能训一次"**：同一基座可挂多个 LoRA，但需注意风格冲突与版权合规。
- **边界**：QLoRA（量化 + LoRA）进一步省显存，属 LLM 微调工程细节，见 [05/003](../05-大语言模型与Transformer/003-预训练与微调.md)。

## 七、一句话总结

- LoRA = $W_0 + BA$，冻结底模、只训低秩补丁；生成式里广泛用于 SD 画风/角色与 LLM 领域适配。
- 省显存、省存储、可合并推理；常与 ControlNet、RAG 等组合。
