# 003 · 多模态 CLIP 基础

> 本文用大白话回答：CLIP 怎么让模型"看懂图又能读懂话"，还把两者对齐到同一套坐标里？为什么搜"一只穿西装的柯基"能找到对的图？
>
> 读完你会知道：CLIP = 用海量图文对训练，把**图片和句子映射到同一向量空间**——相似的图和文靠得近，不相似的离得远。

## 一、一句话先说清

**CLIP（Contrastive Language-Image Pre-training，对比语言-图像预训练）= 用成对的「图片 + 文字描述」做对比学习，把图像编码器和文本编码器训练到同一个嵌入空间里，使语义相关的图文向量相近。**

一句话用途：**图文互搜、零样本分类、给扩散模型提供文本条件**（Stable Diffusion 的文本编码器常用 CLIP）。

## 二、打个比方：照片和说明文字挂同一面墙

想象一面巨大的软木板：

- 每张照片用图钉钉在某一位置；
- 每张说明文字也用图钉钉在**同一面墙**的某个位置；
- 训练目标：**真正配对的图和文，图钉位置要尽量靠近**；不配对的要推远。

之后你可以：
- 拿一句新话去找墙上离它最近的图（**文搜图**）；
- 拿一张新图去找离它最近的文字描述（**图搜文 / 零样本分类**）。

| 生活类比 | CLIP 对应 |
| --- | --- |
| 照片与 caption 成对归档 | 训练数据 (image, text) 对 |
| 配对的挂近一点 | 对比损失拉近正样本 |
| 随机乱配的推远 | 负样本在 batch 内推开 |
| 新图新话也能找邻居 | 推理时算相似度检索 |

## 三、它到底解决什么问题

### 问题 1：传统视觉模型只能认固定类别

ImageNet 分类器输出 1000 类固定标签，无法直接理解"一只穿宇航服的狗"这种**开放词汇**描述。

CLIP 在 4 亿级图文对上预训练后，推理时把类别名或任意句子当**文本 prompt**，与图像算相似度——**无需为该任务重新训练分类头**（零样本分类）。

### 问题 2：怎么训练两个模态到同一空间——对比损失

一个 batch 含 $N$ 对图文 $(I_i, T_i)$。算图像嵌入 $\mathbf{v}_i$、文本嵌入 $\mathbf{t}_i$，并 L2 归一化。

相似度矩阵 $S_{ij} = \mathbf{v}_i^\top \mathbf{t}_j / \tau$（$\tau$ 为温度系数）。

**InfoNCE / 对比损失**（对称）：

$$
L = -\frac{1}{2N}\sum_i \Big( \log \frac{e^{S_{ii}}}{\sum_j e^{S_{ij}}} + \log \frac{e^{S_{ii}}}{\sum_j e^{S_{ji}}} \Big)
$$

> 对齐：对角线 $S_{ii}$ 是正样本（真配对），分母里其他 $j$ 是 batch 内负样本——让正确配对在矩阵里"亮起来"，错误配对"暗下去"。这就是第二节"配对的靠近、乱配的推开"。

### 问题 3：零样本分类怎么用

对类别 $c$，构造文本 prompt 如 `"a photo of a {c}"`，与图像算相似度，取最高者：

$$
\hat{y} = \arg\max_c \; \text{sim}(I, \text{Text}(c))
$$

> 对齐：不用改网络结构，只是把每个类别名变成一句话，看图像向量和哪句话最近——开放词汇分类。

## 四、专业视角（与大白话对齐）

### 4.1 架构

- **图像编码器**：ResNet 或 ViT（见 [06/005 ViT](../06-计算机视觉/005-视觉Transformer ViT.md)）→ 向量 $\mathbf{v}\in\mathbb{R}^d$；
- **文本编码器**：Transformer（见 [05/002](../05-大语言模型与Transformer/002-Transformer架构.md)）→ 向量 $\mathbf{t}\in\mathbb{R}^d$；
- 同一维度 $d$，归一化后点积 = 余弦相似度。

### 4.2 应用

| 应用 | 说明 |
| --- | --- |
| 图文检索 | 以文搜图、以图搜文 |
| 零样本分类 | 用 prompt 列表代替固定 softmax |
| 生成模型条件 | SD 用 CLIP Text Encoder 编码 prompt |
| 数据过滤 | 用 CLIP 分数筛高质量图文对 |

### 4.3 局限

- 细粒度区分（品种、计数）仍弱于专用模型；
- 对 prompt 措辞敏感（需 prompt engineering）；
- 训练数据偏差会带入下游（刻板印象、文化偏见）。

## 五、案例解析：3 张图 3 句话的对比矩阵

Batch $N=3$：

|  | 文本 T1："一只猫" | T2："一条狗" | T3："一辆汽车" |
| --- | --- | --- | --- |
| **图 I1（猫）** | **高** ✓ | 低 | 低 |
| **图 I2（狗）** | 低 | **高** ✓ | 低 |
| **图 I3（汽车）** | 低 | 低 | **高** ✓ |

训练后对角线相似度高、非对角线低。推理时来新图 $I_{\text{new}}$，与三个文本算相似度，最高者对应语义标签——无需见过该任务的标注数据。

## 六、常见误区与边界

- **误区："CLIP 理解语言像 GPT"**：CLIP 文本塔较短，擅长语义对齐而非长文推理；复杂推理仍靠 LLM。
- **误区："零样本一定够用"**：专业领域（医疗影像）往往仍需微调或专用模型。
- **边界**：OpenCLIP、SigLIP 等为开源/改进版；多模态大模型（GPT-4V、LLaVA）在 CLIP 思想上扩展为"对话式视觉"。

## 七、一句话总结

- CLIP = 图文双塔 + 对比学习，共享嵌入空间；支撑零样本分类与文生图条件编码。
