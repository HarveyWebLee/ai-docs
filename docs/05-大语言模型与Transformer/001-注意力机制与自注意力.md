# 001 · 注意力机制与自注意力

> 本文回答：注意力机制到底在"注意"什么？Query/Key/Value 是什么？缩放点积注意力为什么要除以 $\sqrt{d_k}$？

## 一、直觉与通俗解读

读到句子"**它**很好吃"时，你会自动回看前文，判断"它"指代的是"苹果"还是"椅子"。这种"**读当前词时，自动关注其它相关词**"的能力，就是注意力机制。

一个更贴切的类比是"**查字典**"：

- 你手上有一个**问题**（Query，如"它指代谁"）；
- 字典里每个词条有一个**索引/关键词**（Key）和对应的**内容**（Value）；
- 你把 Query 和每个 Key 比对，算出"相关度"，然后按相关度加权汇总各词条的 Value，得到答案。

**自注意力**就是让句子里"每个词都对其它所有词做一次这样的查字典"，从而捕捉词与词之间的依赖关系。

## 二、严谨定义与原理

### 2.1 缩放点积注意力（Scaled Dot-Product Attention）

给定查询矩阵 $Q$、键矩阵 $K$、值矩阵 $V$：

$$
\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

- $QK^\top$：每个 Query 与每个 Key 的点积，衡量相关度（注意力分数）。
- $\sqrt{d_k}$：**缩放因子**（$d_k$ 为 Key 维度）。当维度较大时，点积数值会变大，使 softmax 进入梯度极小的饱和区；除以 $\sqrt{d_k}$ 让方差稳定，训练更稳。
- $\mathrm{softmax}$：把分数归一化为权重（和为 1）。
- 最后用权重对 $V$ 加权求和。

### 2.2 自注意力（Self-Attention）

自注意力中，$Q,K,V$ 都由**同一输入序列**经不同线性变换得到：$Q=XW^Q,\ K=XW^K,\ V=XW^V$。因此序列中每个位置都能直接"看到"任意其它位置——这是它相比 RNN 的关键优势：**不受距离限制、可并行计算**。

### 2.3 多头注意力（Multi-Head）

把注意力并行做 $h$ 次（每个"头"用不同的投影），再拼接：

$$
\mathrm{MultiHead}(Q,K,V) = \mathrm{Concat}(\text{head}_1,\dots,\text{head}_h)W^O
$$

不同的头可以关注不同类型的关系（如语法依赖、指代、局部搭配），增强表达力。

## 三、案例解析：一次注意力权重的计算

设某 Query 与三个 Key 的点积分数（已除以 $\sqrt{d_k}$）为 $[2, 1, 0]$。softmax 归一化：

$$
\text{权重} = \frac{[e^2, e^1, e^0]}{e^2+e^1+e^0} = \frac{[7.389, 2.718, 1.000]}{11.107} \approx [0.665, 0.245, 0.090]
$$

于是输出 $= 0.665\,V_1 + 0.245\,V_2 + 0.090\,V_3$：**得分最高的第 1 个词条贡献最大**。这正是"按相关度加权汇总信息"的直观体现。

## 四、常见误区与边界

- **误区："注意力就是 RNN 的改进版循环"**：注意力**无循环**，直接建模任意两位置的关系，可完全并行。
- **误区："多头就是把维度切大"**：多头的价值在于**多子空间关注不同关系**，不仅是维度切分。
- **复杂度边界**：自注意力对序列长度 $n$ 的计算/显存为 $O(n^2)$，长序列成本高，催生了各种高效注意力变体（稀疏、线性、FlashAttention 等）。

## 五、小结与延伸阅读

- 注意力 = 用 Query 与 Key 算相关度、对 Value 加权汇总（"查字典"）。
- 自注意力让每个词直接关联全序列，可并行、无长距离衰减；多头捕捉多种关系。
- 缩放 $1/\sqrt{d_k}$ 稳定训练；复杂度 $O(n^2)$ 是长文本的主要瓶颈。
