# 002 · 目录与命名规范

> 本文定义仓库的目录结构、文件命名与配套代码的组织方式，是保证"系统性分类"的基础设施。

## 一、顶层目录

```text
ai-docs/
├── docs/                     # 所有学习文档（VitePress 站点根目录）
│   ├── .vitepress/           # 站点配置（config、侧边栏生成器）
│   ├── index.md              # 站点首页
│   ├── 00-元规范/            # 文档规范分类
│   ├── 01-数学与理论基础/
│   ├── 03-深度学习基础/      # 示范分类
│   └── NN-分类名/            # 其它系统性分类
├── code/                     # 配套学习/测试代码，按分类镜像组织
│   └── 03-深度学习基础/
├── scripts/                  # 校验脚本等工具
└── AGENTS.md                 # 面向 AI Agent 的仓库约定
```

## 二、分类目录命名

- 格式：`两位数序号-分类名`，例如 `03-深度学习基础`。
- 序号决定侧边栏中分类的排列顺序，序号应体现学习路径的先后依赖。
- 分类名使用简体中文，简洁准确，不含空格与特殊符号。

## 三、文档文件命名

- 格式：`三位数序号-知识点名.md`，例如 `001-神经网络结构.md`。
- **每个分类必须包含 `000-分类总览与知识图谱.md`**：作为该分类的导读、知识串联与知识图谱页，序号 `000` 保证其排在最前。
- 其余知识点从 `001` 起顺序编号；序号体现该分类内部的阅读顺序。
- 序号预留间隔：如需在已有文档间插入内容，可使用未占用的序号（如在 `001` 与 `002` 之间新增时，可整体重排或使用中间号），保持顺序清晰。

## 四、配套代码组织

- 学习/测试代码统一放在 `code/` 下，并**镜像文档分类目录**，例如：
  - 文档：`docs/03-深度学习基础/001-神经网络结构.md`
  - 代码：`code/03-深度学习基础/001-神经网络结构/`（可含 `main.py`、`README.md` 等）
- 文档中引用代码时使用相对路径链接，保证可追溯。

## 五、命名校验

仓库提供校验脚本，随时可运行：

```bash
npm run docs:validate
```

它会检查：

1. `docs/` 下每个分类目录名是否符合 `NN-分类名`。
2. 每个分类内的 `.md` 是否以三位序号开头。
3. 每个分类是否包含 `000-` 开头的总览/知识图谱文档。

校验也会由 Git `pre-commit` 钩子在提交前自动触发（见仓库 `.husky/` 或 `.githooks/` 配置）。

## 六、小结

- 文档放 `docs/NN-分类名/NNN-知识点.md`，代码放 `code/` 下镜像目录。
- 每个分类必须有 `000-` 总览与知识图谱文档。
- 命名规范由 `npm run docs:validate` 与 Git 钩子共同保障。
