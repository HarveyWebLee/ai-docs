## 三、01 · 数学与理论基础

### 向量（Vector）

通俗：把一件事用**一串有顺序的数**表示，像「身高、体重、年龄」排成 `[175, 70, 28]`；AI 里一条样本、一张图展平后都是向量。
专业：$\mathbf{x} \in \mathbb{R}^n$，$n$ 维实数列；点积 $\mathbf{a}\!\cdot\!\mathbf{b}$ 衡量方向一致性，范数 $\|\mathbf{x}\|$ 衡量「长度/大小」。
用法：见 [01/001 线性代数](../01-数学与理论基础/001-线性代数基础.md)、[03/001 神经网络结构](../03-深度学习基础/001-神经网络结构.md)。

### 矩阵（Matrix）

通俗：一台「**批量加工向量**」的机器，输入一串数、输出另一串数；神经网络每一层本质都是矩阵乘法。
专业：$A \in \mathbb{R}^{m \times n}$；$A\mathbf{x}$ 为线性变换，$AB$ 表示复合变换（一般 $AB \neq BA$）。
用法：见 [01/001 线性代数](../01-数学与理论基础/001-线性代数基础.md)。

### 特征值与特征向量（Eigenvalue & Eigenvector）

通俗：矩阵作用在某些特殊方向上**只拉长/压短、不改变方向**，这些方向叫特征向量，拉长倍数叫特征值。
专业：$A\mathbf{v} = \lambda \mathbf{v}$；PCA 用协方差矩阵最大特征值对应方向作主成分。
用法：见 [01/001](../01-数学与理论基础/001-线性代数基础.md)、[02/005 决策树](../02-机器学习基础/005-决策树与集成方法.md)（基尼 vs 信息增益背景）。

### 奇异值分解（Singular Value Decomposition，SVD）

通俗：把任意矩阵拆成「旋转 → 拉伸 → 再旋转」三步，抓住最重要的几个拉伸方向就能**大幅压缩信息**。
专业：$A = U\Sigma V^\top$；低秩近似、推荐系统、降维的核心工具。
用法：见 [01/001 线性代数](../01-数学与理论基础/001-线性代数基础.md)。

### 梯度（Gradient）

通俗：多元函数在某点**上升最快的方向**；训练时要**反着走**（负梯度），像蒙眼下山找最低点。
专业：$\nabla f = (\partial f/\partial x_1, \ldots)$；无约束极值必要条件 $\nabla f(\mathbf{x}^*)=\mathbf{0}$。
用法：见 [01/003 微积分与梯度](../01-数学与理论基础/003-微积分与梯度.md)、[03/002 反向传播](../03-深度学习基础/002-反向传播与梯度下降.md)。

### 链式法则（Chain Rule）

通俗：「A 影响 B、B 影响 C」时，A 对 C 的影响 = **两段影响相乘**；反向传播就是链式法则层层相乘。
专业：$\frac{\partial L}{\partial w} = \frac{\partial L}{\partial y}\frac{\partial y}{\partial w}$；多层网络梯度高效计算的基础。
用法：见 [01/003](../01-数学与理论基础/003-微积分与梯度.md)、[03/002](../03-深度学习基础/002-反向传播与梯度下降.md)。

### 熵（Entropy，$H$）

通俗：衡量「**结果有多不确定**」——抛硬币结果难猜，熵高；确定事件熵低。
专业：离散分布 $H(p) = -\sum_i p_i \log p_i$；单位通常为 nat 或 bit。
用法：见 [01/004 信息论](../01-数学与理论基础/004-信息论基础.md)、[02/005 决策树](../02-机器学习基础/005-决策树与集成方法.md)（信息增益）。

### 交叉熵（Cross-entropy）

通俗：衡量「**预测分布离真实答案有多远**」；分类任务里猜得越准，交叉熵越小。
专业：二分类 $L = -[y\log\hat{p} + (1-y)\log(1-\hat{p})]$；等价于负对数似然。
用法：见 [01/004](../01-数学与理论基础/004-信息论基础.md)、[02/002 逻辑回归](../02-机器学习基础/002-线性回归与逻辑回归.md)、[03/001 神经网络](../03-深度学习基础/001-神经网络结构.md)。

### KL 散度（Kullback–Leibler Divergence，KL）

通俗：两个概率分布「**差多少**」的非对称距离；RLHF 里用来防止新策略偏离旧模型太远。
专业：$D_{\mathrm{KL}}(P\|Q) = \sum_i P(i)\log\frac{P(i)}{Q(i)}$；非对称，非严格距离。
用法：见 [01/004](../01-数学与理论基础/004-信息论基础.md)、[05/005 对齐与 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)。

### 梯度下降（Gradient Descent，GD）

通俗：每次沿**负梯度**挪一小步，让损失慢慢变小。
专业：$\theta \leftarrow \theta - \eta \nabla J(\theta)$；$\eta$ 为学习率。
用法：见 [01/005 最优化](../01-数学与理论基础/005-最优化基础.md)、[03/002](../03-深度学习基础/002-反向传播与梯度下降.md)。

### 随机梯度下降（Stochastic Gradient Descent，SGD）

通俗：不用全部数据算梯度，**每次抽一条或一小批**，算得快、方向有点抖，但能帮跳出坏坑。
专业：$\theta$ 用 mini-batch 梯度估计更新；工业界默认小批量 SGD 或其变体。
用法：见 [01/005](../01-数学与理论基础/005-最优化基础.md)、[03/003 优化器](../03-深度学习基础/003-优化器与学习率调度.md)。

### 动量（Momentum）

通俗：更新时**带上历史方向的惯性**，在狭长山谷里少左右横跳、沿谷底加速。
专业：$v \leftarrow \beta v + \nabla J$，$\theta \leftarrow \theta - \eta v$；$\beta \approx 0.9$。
用法：见 [01/005](../01-数学与理论基础/005-最优化基础.md)、[03/003](../03-深度学习基础/003-优化器与学习率调度.md)。

### Adam（Adaptive Moment Estimation，Adam）

通俗：给**每个参数自动调步子大小**，还自带动量；深度学习「开箱即用」首选优化器之一。
专业：维护梯度一阶矩 $m$、二阶矩 $v$，偏差修正后更新；默认 $\beta_1=0.9,\beta_2=0.999$。
用法：见 [01/005](../01-数学与理论基础/005-最优化基础.md)、[03/003](../03-深度学习基础/003-优化器与学习率调度.md)。

### 凸优化（Convex Optimization）

通俗：损失面像**光滑大碗**，只有一个坑，滚到底就一定是全局最低。
专业：凸函数任意局部极小即全局极小；深度学习损失通常**非凸**。
用法：见 [01/005 最优化](../01-数学与理论基础/005-最优化基础.md)。

### 贝叶斯定理（Bayes' Theorem）

通俗：已知新证据后，**更新对某事的相信程度**——「看到阳性结果，真患病的概率是多少」。
专业：$P(A|B)=\frac{P(B|A)P(A)}{P(B)}$；先验 × 似然 → 后验。
用法：见 [01/002 概率与统计](../01-数学与理论基础/002-概率与统计基础.md)、[02/002 逻辑回归](../02-机器学习基础/002-线性回归与逻辑回归.md)（垃圾邮件案例）。

### 条件概率（Conditional Probability）

通俗：在**已经知道 B 发生**的前提下，A 发生的概率。
专业：$P(A|B)=P(A,B)/P(B)$；分类、生成模型、RL 均依赖条件分布。
用法：见 [01/002](../01-数学与理论基础/002-概率与统计基础.md)。

### 先验 / 后验 / 似然（Prior / Posterior / Likelihood）

通俗：**先验**=没看数据前的猜测；**似然**=数据在假设下有多常见；**后验**=结合数据后的更新判断。
专业：贝叶斯推断三要素；MLE 最大化似然，MAP 加先验。
用法：见 [01/002](../01-数学与理论基础/002-概率与统计基础.md)。

### 期望与方差（Expectation & Variance）

通俗：**期望**=长期平均结果；**方差**=结果波动有多大、稳不稳。
专业：$E[X]=\sum x p(x)$，$\mathrm{Var}(X)=E[(X-E[X])^2]$；BatchNorm 用 batch 统计估计。
用法：见 [01/002](../01-数学与理论基础/002-概率与统计基础.md)、[03/004 BatchNorm](../03-深度学习基础/004-正则化与批归一化.md)。

### 高斯分布（Gaussian / Normal Distribution）

通俗：自然界最常见的**钟形曲线**——多数值集中在中间，离均值越远越少。
专业：$\mathcal{N}(\mu,\sigma^2)$；噪声建模、初始化、扩散前向过程基础。
用法：见 [01/002](../01-数学与理论基础/002-概率与统计基础.md)、[10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)。

### 最大似然估计（Maximum Likelihood Estimation，MLE）

通俗：选一组参数，让**现有数据出现的可能性最大**——「哪种参数最像这些数据生成的」。
专业：$\hat\theta=\arg\max_\theta \prod_i P(x_i|\theta)$；交叉熵最小化等价于 MLE（分类）。
用法：见 [01/002](../01-数学与理论基础/002-概率与统计基础.md)、[01/004 信息论](../01-数学与理论基础/004-信息论基础.md)。

## 四、02 · 机器学习基础

### 监督学习（Supervised Learning）

通俗：给模型看**带标准答案**的数据，让它学会「输入 → 输出」的映射。
专业：学习 $P(y|x)$ 或 $f:x\to y$；分**分类**（离散标签）与**回归**（连续值）。
用法：见 [02/001 监督与无监督学习](../02-机器学习基础/001-监督学习与无监督学习.md)。

### 无监督学习（Unsupervised Learning）

通俗：只有题、**没有答案**，让模型自己找规律（分堆、降维）。
专业：聚类、密度估计、降维、生成建模等；评估无统一标签。
用法：见 [02/001](../02-机器学习基础/001-监督学习与无监督学习.md)。

### 泛化能力（Generalization）

通俗：模型在**没见过的题**上也能答对，而不是只会背训练集。
专业：测试集期望风险；过拟合时训练好、泛化差。
用法：见 [02/001](../02-机器学习基础/001-监督学习与无监督学习.md)、[02/004 过拟合](../02-机器学习基础/004-过拟合与正则化.md)。

### 过拟合（Overfitting）

通俗：**死记硬背**训练集（连噪声都背），一考新题就露馅。
专业：高方差、模型复杂度过高；训练误差低、验证/测试误差高。
用法：见 [02/004](../02-机器学习基础/004-过拟合与正则化.md)、[03/004 正则化](../03-深度学习基础/004-正则化与批归一化.md)。

### 欠拟合（Underfitting）

通俗：模型太笨，**连训练集都学不好**。
专业：高偏差；增加容量、特征或训练时间可缓解。
用法：见 [02/004](../02-机器学习基础/004-过拟合与正则化.md)。

### 偏差-方差权衡（Bias-Variance Tradeoff）

通俗：**太简单学不好（偏差高），太复杂背噪声（方差高）**；要在中间找甜点。
专业：期望误差分解为偏差² + 方差 + 不可约噪声。
用法：见 [02/004](../02-机器学习基础/004-过拟合与正则化.md)。

### 正则化（Regularization）

通俗：给模型**立规矩**，别让它学太野；L2 压权重、Dropout 随机「请假」都是正则。
专业：在目标中加入惩罚项或约束，降低有效复杂度。
用法：见 [02/004](../02-机器学习基础/004-过拟合与正则化.md)、[03/004](../03-深度学习基础/004-正则化与批归一化.md)。

### 混淆矩阵（Confusion Matrix）

通俗：分类结果的**四格账单**：猜对/猜错各多少，是算精确率、召回率的基础。
专业：TP/FP/TN/FN 四格；**真正例 TP**=有病判有病，**假正例 FP**=没病判有病（误报），**假负例 FN**=有病判没病（漏诊）。
用法：见 [02/003 模型评估](../02-机器学习基础/003-模型评估与指标.md)；配套代码 `code/02-机器学习基础/003-模型评估与指标/eval_metrics_demo.py`。

### 精确率（Precision）

通俗：**你报警的人里，真有问题的占多少**——少误报看这个。
专业：$P = TP/(TP+FP)$。
用法：见 [02/003](../02-机器学习基础/003-模型评估与指标.md)。

### 召回率（Recall）

通俗：**该抓的人里，你抓到了多少**——少漏诊看这个。
专业：$R = TP/(TP+FN)$；又称灵敏度、真阳性率 TPR。
用法：见 [02/003](../02-机器学习基础/003-模型评估与指标.md)。

### F1 分数（F1 Score）

通俗：精确率和召回率的**平衡分**，两个都高 F1 才高。
专业：$F_1 = 2PR/(P+R)$，调和平均。
用法：见 [02/003](../02-机器学习基础/003-模型评估与指标.md)。

### AUC（Area Under the ROC Curve，AUC）

通俗：不挑阈值，看模型**整体排序能力**；0.5 像瞎猜，1 完美。
专业：ROC 曲线（TPR vs FPR）下面积；对类别不平衡较稳健。
用法：见 [02/003](../02-机器学习基础/003-模型评估与指标.md)。

### K 折交叉验证（K-fold Cross-validation）

通俗：把数据分 K 份，**轮流拿一份当考卷**，其余当练习册，取平均成绩更稳。
专业：减少划分偶然性；小数据集常用。
用法：见 [02/003](../02-机器学习基础/003-模型评估与指标.md)。

### 决策树（Decision Tree）

通俗：像玩「二十个问题」，每次选一个特征问阈值，把数据越分越纯。
专业：CART 等贪心递归分裂；路径即 if-else 规则，可解释性强。
用法：见 [02/005 决策树与集成](../02-机器学习基础/005-决策树与集成方法.md)。

### 随机森林（Random Forest，RF）

通俗：很多棵决策树**各自看不同样本、不同特征**，最后**投票**——更稳、不易过拟合。
专业：Bagging + 随机子空间；分类多数票，回归取平均。
用法：见 [02/005](../02-机器学习基础/005-决策树与集成方法.md)。

### XGBoost（eXtreme Gradient Boosting，XGBoost）

通俗：一棵树接一棵树，**后面的树专门纠正前面错的**，表格数据竞赛常胜。
专业：GBDT + 二阶泰勒 + 正则；串行训练，对缺失值友好。
用法：见 [02/005](../02-机器学习基础/005-决策树与集成方法.md)。

### 主成分分析（Principal Component Analysis，PCA）

通俗：把高维数据**压到少数几个「最主要方向」**，丢掉次要波动。
专业：协方差矩阵特征向量；无监督线性降维。
用法：见 [02/001](../02-机器学习基础/001-监督学习与无监督学习.md)、[01/001 线性代数](../01-数学与理论基础/001-线性代数基础.md)。

### 线性回归（Linear Regression）

通俗：用**一条直线（或超平面）**拟合「输入 → 连续数值输出」，如房价预测。
专业：$\hat{y}=\mathbf{w}^\top\mathbf{x}+b$；最小化 MSE；单层线性网络等价形式。
用法：见 [02/002 线性回归与逻辑回归](../02-机器学习基础/002-线性回归与逻辑回归.md)；配套 `code/02-机器学习基础/002-线性回归与逻辑回归/linear_regression.py`。

### 逻辑回归（Logistic Regression）

通俗：线性回归的**分类版**——输出 0~1 概率，再按 0.5 阈值分两类。
专业：$\hat{p}=\sigma(\mathbf{w}^\top\mathbf{x}+b)$；交叉熵损失；单层神经网络形态。
用法：见 [02/002](../02-机器学习基础/002-线性回归与逻辑回归.md)。

### 均方误差（Mean Squared Error，MSE）

通俗：预测值与真值**差多少的平方再平均**；差越大罚越重。
专业：$L=\frac{1}{n}\sum(\hat{y}-y)^2$；回归标准损失；对异常值敏感。
用法：见 [02/002](../02-机器学习基础/002-线性回归与逻辑回归.md)、[03/002 损失函数](../03-深度学习基础/002-反向传播与梯度下降.md)。

### 准确率（Accuracy）

通俗：**猜对的占全部样本的比例**；类别均衡时直观，不平衡时易误导。
专业：$(TP+TN)/(TP+TN+FP+FN)$；与 P/R/F1 互补使用。
用法：见 [02/003 模型评估](../02-机器学习基础/003-模型评估与指标.md)。

### ROC 曲线（Receiver Operating Characteristic Curve，ROC）

通俗：不断调整分类阈值，画出**真阳性率 vs 假阳性率**的曲线；曲线越靠左上越好。
专业：横轴 FPR、纵轴 TPR；AUC 为其下面积。
用法：见 [02/003](../02-机器学习基础/003-模型评估与指标.md)（与 AUC 词条配合阅读）。

### L1 / L2 正则（L1 / L2 Regularization）

通俗：L2 **把权重往小压**（权重大就罚）；L1 **鼓励很多权重变 0**（自动做特征选择）。
专业：L2：$\lambda\|\mathbf{w}\|_2^2$；L1：$\lambda\|\mathbf{w}\|_1$；权重衰减常指 L2。
用法：见 [02/004 过拟合与正则化](../02-机器学习基础/004-过拟合与正则化.md)、[03/003 AdamW](../03-深度学习基础/003-优化器与学习率调度.md)。

### 基尼不纯度（Gini Impurity）

通俗：随机抽两个样本**类别不一致的概率**；越纯（同一类）基尼越小。
专业：$G=1-\sum_k p_k^2$；CART 决策树分裂准则之一。
用法：见 [02/005 决策树与集成](../02-机器学习基础/005-决策树与集成方法.md)。

### 信息增益（Information Gain，IG）

通俗：按某特征分裂后，**不确定性减少了多少**；减得越多越值得分裂。
专业：$IG=H(\text{父})-\sum \frac{n_j}{n}H(\text{子}_j)$；与熵配合。
用法：见 [02/005](../02-机器学习基础/005-决策树与集成方法.md)。

### Bagging（Bootstrap Aggregating，Bagging）

通俗：**有放回抽样**训练多个模型，最后投票/平均——降方差、抗过拟合。
专业：Bootstrap 采样 + 聚合；随机森林 = Bagging + 决策树。
用法：见 [02/005](../02-机器学习基础/005-决策树与集成方法.md)。

### Boosting

通俗：模型**串行接力**，后面的专门纠正前面错的样本——降偏差、提精度。
专业：AdaBoost、GBDT、XGBoost 均属 Boosting 族；与 Bagging 并行集成相对。
用法：见 [02/005](../02-机器学习基础/005-决策树与集成方法.md)。

### 梯度提升决策树（Gradient Boosting Decision Tree，GBDT）

通俗：每加一棵树，都沿着**当前损失的梯度方向**去拟合残差。
专业：Boosting + 决策树；XGBoost/LightGBM 为其工程优化版。
用法：见 [02/005](../02-机器学习基础/005-决策树与集成方法.md)。

### K-Means 聚类（K-Means Clustering）

通俗：指定 K 个中心，样本**归到最近中心**，再更新中心，反复直到稳定——自动分 K 堆。
专业：无监督聚类；最小化簇内平方和；需预设 K、对初始化敏感。
用法：见 [02/001 监督与无监督](../02-机器学习基础/001-监督学习与无监督学习.md)。

## 五、03 · 深度学习基础

### 神经网络（Neural Network）

通俗：很多「打分员」分层排队，每层对数字加工，最后一层给答案。
专业：由神经元、层、连接权重与激活函数组成；前馈网络为 DAG。
用法：见 [03/001 神经网络结构](../03-深度学习基础/001-神经网络结构.md)。

### 激活函数（Activation Function）

通俗：决定神经元**要不要吭声、吭多大声**；没有它多层网络退化成一层直线。
专业：非线性 $\sigma$；ReLU、Sigmoid、Tanh 等；引入表达非线性映射的能力。
用法：见 [03/001](../03-深度学习基础/001-神经网络结构.md)（黄金样板节）。

### ReLU（Rectified Linear Unit，ReLU）

通俗：负数归零，正数原样——像「不够格就闭嘴」。
专业：$\max(0,z)$；缓解梯度消失，计算简单；Dead ReLU 为已知问题。
用法：见 [03/001](../03-深度学习基础/001-神经网络结构.md)、[06/002 CNN](../06-计算机视觉/002-卷积神经网络CNN.md)。

### 反向传播（Backpropagation，Backprop）

通俗：从输出误差**倒着传**，一次算出每个权重该怎么调；不是训练算法本身，是**算梯度的方法**。
专业：链式法则在计算图上的实现；与梯度下降配合完成训练。
用法：见 [03/002 反向传播与梯度下降](../03-深度学习基础/002-反向传播与梯度下降.md)。

### 损失函数（Loss Function）

通俗：衡量「**预测有多差**」的分数，越低越好。
专业：MSE（回归）、交叉熵（分类）等；训练目标 $\min_\theta J(\theta)$。
用法：见 [03/002](../03-深度学习基础/002-反向传播与梯度下降.md)。

### 学习率（Learning Rate，$\eta$）

通俗：梯度下降每步**迈多大**；太大震荡，太小磨蹭。
专业：关键超参数；常配合 warmup、余弦退火等调度。
用法：见 [03/003 优化器与学习率调度](../03-深度学习基础/003-优化器与学习率调度.md)。

### AdamW（Adam with Weight Decay，AdamW）

通俗：Adam 的升级版，把**权重衰减**从梯度里拆出来；Transformer 训练常用。
专业：解耦 L2 正则与自适应学习率；LLM/SD 微调默认之一。
用法：见 [03/003](../03-深度学习基础/003-优化器与学习率调度.md)、[05/003 预训练与微调](../05-大语言模型与Transformer/003-预训练与微调.md)。

### Dropout

通俗：训练时**随机让一部分神经元「请假」**，逼网络别依赖少数节点，减轻过拟合。
专业：以概率 $p$ 置零激活，推理时缩放或等价合并。
用法：见 [03/004 正则化与批归一化](../03-深度学习基础/004-正则化与批归一化.md)。

### 批归一化（Batch Normalization，BatchNorm）

通俗：每一层输入先**拉成均值 0、方差 1 附近**，训练更稳、可用更大学习率。
专业：对 mini-batch 统计 $\mu,\sigma$ 归一化，再学 $\gamma,\beta$；推理用滑动平均。
用法：见 [03/004](../03-深度学习基础/004-正则化与批归一化.md)、[06/002 CNN](../06-计算机视觉/002-卷积神经网络CNN.md)。

### 层归一化（Layer Normalization，LayerNorm）

通俗：在**单个样本的所有特征**上归一化，不依赖 batch 大小；Transformer 标配。
专业：对特征维求均值方差；Pre-LN / Post-LN 为放置位置差异。
用法：见 [03/004](../03-深度学习基础/004-正则化与批归一化.md)、[05/002 Transformer](../05-大语言模型与Transformer/002-Transformer架构.md)。

### 梯度消失 / 梯度爆炸（Gradient Vanishing / Exploding）

通俗：网络太深时，梯度传回去**越来越弱或越来越强**，前面层学不动或炸掉。
专业：深层连乘 Jacobian 谱半径问题；ReLU、残差、归一化、裁剪缓解。
用法：见 [03/002](../03-深度学习基础/002-反向传播与梯度下降.md)、[06/003 ResNet](../06-计算机视觉/003-经典CNN架构与残差网络.md)。

### 残差连接（Residual Connection / Skip Connection）

通俗：给网络加**抄近道**，这层可以「啥也不干原样传下去」，深层才好训。
专业：$y = F(x) + x$；ResNet 核心，缓解退化与梯度消失。
用法：见 [06/003 ResNet](../06-计算机视觉/003-经典CNN架构与残差网络.md)、[05/002 Transformer](../05-大语言模型与Transformer/002-Transformer架构.md)。

### 前向传播（Forward Propagation）

通俗：数据从输入层**一层层算到输出**，得到预测结果；训练时与反向传播成对出现。
专业：按计算图拓扑序执行；推理阶段仅前向。
用法：见 [03/002 反向传播](../03-深度学习基础/002-反向传播与梯度下降.md)、[03/000 总览](../03-深度学习基础/000-分类总览与知识图谱.md)。

### Sigmoid 激活函数（Sigmoid Activation）

通俗：把任意实数**压到 0~1**，像概率开关；负数趋 0、正数趋 1。
专业：$\sigma(z)=1/(1+e^{-z})$；易梯度消失；逻辑回归输出层、二分类常用。
用法：见 [03/001 神经网络结构](../03-深度学习基础/001-神经网络结构.md)、[02/002 逻辑回归](../02-机器学习基础/002-线性回归与逻辑回归.md)。

### Tanh 激活函数（Hyperbolic Tangent，Tanh）

通俗：Sigmoid 的**对称版**，输出 -1~1，零附近更敏感。
专业：$\tanh(z)$；RNN/LSTM 历史常用；仍可能梯度消失。
用法：见 [03/001](../03-深度学习基础/001-神经网络结构.md)。

### 学习率调度（Learning Rate Schedule）

通俗：训练过程中**动态改学习率**——开头小步探路（Warmup），后期小步精调（Cosine 衰减）。
专业：Step decay、Cosine annealing、Warmup 等；大模型训练标配。
用法：见 [03/003 优化器与学习率调度](../03-深度学习基础/003-优化器与学习率调度.md)。

### 学习率预热（Learning Rate Warmup）

通俗：训练开头**先用很小的学习率**，再逐渐升到目标值，避免一开始步子太大炸掉。
专业：前 $T_w$ 步线性/余弦升温；Transformer/LLM 预训练几乎必用。
用法：见 [03/003](../03-深度学习基础/003-优化器与学习率调度.md)。

### 权重衰减（Weight Decay）

通俗：每步更新时**把权重往 0 拉一点**，防止权重无限变大、减轻过拟合。
专业：等价于 L2 正则（在 SGD 中）；AdamW 将其与自适应步长解耦。
用法：见 [03/003](../03-深度学习基础/003-优化器与学习率调度.md)、[03/004 正则化](../03-深度学习基础/004-正则化与批归一化.md)。

### 早停（Early Stopping）

通俗：验证集成绩**不再变好就停训**，别继续练到过拟合。
专业：监控 validation loss；保存最佳 checkpoint。
用法：见 [03/004](../03-深度学习基础/004-正则化与批归一化.md)、[02/004 过拟合](../02-机器学习基础/004-过拟合与正则化.md)。

### 数据增强（Data Augmentation）

通俗：训练时对图像**随机翻转、裁剪、变色**等，造更多「变体」，让模型见多识广。
专业：不改变标签的输入变换；CV 正则与泛化核心手段。
用法：见 [03/004](../03-深度学习基础/004-正则化与批归一化.md)、[06/002 CNN](../06-计算机视觉/002-卷积神经网络CNN.md)。

## 六、04 · 自然语言处理

### 自然语言处理（Natural Language Processing，NLP）

通俗：让计算机**理解、生成人类语言**的技术总称。
专业：从规则系统到统计 NLP 再到深度学习 NLP、大模型时代。
用法：见 [04/000 总览](../04-自然语言处理/000-分类总览与知识图谱.md)。

### 词向量 / 词嵌入（Word Embedding）

通俗：把词变成**一串数字**，意思相近的词数字也相近。
专业：Word2Vec、GloVe 等；分布式表示，维数 $d$ 通常 100~300。
用法：见 [04/001 文本表示与词向量](../04-自然语言处理/001-文本表示与词向量.md)。

### 分词 / 词元（Tokenization / Token）

通俗：把句子切成模型能处理的**小块**（字、词或子词）。
专业：Token 为模型最小输入单元；中文需专门分词或子词算法。
用法：见 [04/002 分词与子词切分](../04-自然语言处理/002-分词与子词切分.md)。

### 字节对编码（Byte Pair Encoding，BPE）

通俗：从字符开始，**反复合并最高频相邻对**，造出子词词表；GPT 系常用。
专业：数据驱动子词切分；平衡 OOV 与词表大小。
用法：见 [04/002](../04-自然语言处理/002-分词与子词切分.md)。

### 循环神经网络（Recurrent Neural Network，RNN）

通俗：按顺序读序列，**带着「记忆」**处理每个词；适合文本但难并行。
专业：隐状态 $h_t = f(h_{t-1}, x_t)$；长依赖弱、梯度消失。
用法：见 [04/003 RNN 与 LSTM](../04-自然语言处理/003-循环神经网络RNN与LSTM.md)。

### 长短期记忆网络（Long Short-Term Memory，LSTM）

通俗：RNN 升级版，有**遗忘门、输入门、输出门**，记忆更可控。
专业：门控机制缓解长依赖；BiLSTM 双向读序列。
用法：见 [04/003](../04-自然语言处理/003-循环神经网络RNN与LSTM.md)。

### 命名实体识别（Named Entity Recognition，NER）

通俗：在句子里**圈出人名、地名、机构名**等，是逐词标注不是整句一个标签。
专业：序列标注；BIO 方案标记实体边界。
用法：见 [04/004 文本分类与 NER](../04-自然语言处理/004-文本分类与命名实体识别.md)。

### 语言模型（Language Model，LM）

通俗：给前面文字，猜**下一个词是什么**；ChatGPT 本质就是超大语言模型。
专业：建模 $P(w_t|w_{<t})$；自回归训练最大化似然。
用法：见 [04/005 文本生成](../04-自然语言处理/005-文本生成基础.md)、[05/003 预训练](../05-大语言模型与Transformer/003-预训练与微调.md)。

### 自回归生成（Autoregressive Generation）

通俗：**接龙**：生成一个词，拼到后面，再预测下一个，循环直到结束。
专业：Decoder-only 模型推理范式；与扩散「一步 denoise」不同。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)、[08/005 成本优化](../08-AI工程与MLOps/005-成本与性能优化.md)（KV Cache）。

### 束搜索（Beam Search，Beam）

通俗：每步保留**多条最有希望的接龙路径**，比贪心更稳，计算量 ×束宽。
专业：宽度 $B$ 的启发式搜索；机器翻译常用，开放对话常用采样。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)。

### 温度（Temperature，$T$）

通俗：控制采样**有多随机**；低温度保守，高温度更有创意也可能胡言。
专业：logits 除以 $T$ 再 softmax；$T\to 0$ 趋近贪心。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)、[09/001 提示工程](../09-提示工程与Agent开发/001-提示工程基础.md)。

### Top-p 采样（Nucleus Sampling，Top-p）

通俗：只在**累计概率达到 p 的最小词集合**里随机抽，避免抽到极低概率怪词。
专业：又称 nucleus sampling；与 Top-k 常配合 temperature。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)。

### TF-IDF（Term Frequency–Inverse Document Frequency，TF-IDF）

通俗：一个词**在本篇出现多、在全库出现少** → 权重高，更能代表这篇文档主题。
专业：$tf\text{-}idf(t,d)=tf(t,d)\times \log\frac{N}{df(t)}$；经典稀疏文本表示。
用法：见 [04/001 文本表示与词向量](../04-自然语言处理/001-文本表示与词向量.md)。

### 词袋模型（Bag-of-Words，BoW）

通俗：把文档看成**一堆词的 multiset**，不管顺序，只数每个词出现几次。
专业：高维稀疏向量；简单但丢失语序。
用法：见 [04/001](../04-自然语言处理/001-文本表示与词向量.md)。

### 独热编码（One-hot Encoding）

通俗：每个词对应**一个超长向量**，只有自己的位置是 1，其余全是 0。
专业：维度 = 词表大小；无法表达语义相似性，词向量为其改进。
用法：见 [04/001](../04-自然语言处理/001-文本表示与词向量.md)。

### 门控循环单元（Gated Recurrent Unit，GRU）

通俗：LSTM 的**简化版**，门更少、参数更少，效果常接近 LSTM。
专业：更新门 + 重置门；隐状态递推；比 LSTM 略快。
用法：见 [04/003 RNN 与 LSTM](../04-自然语言处理/003-循环神经网络RNN与LSTM.md)。

### 双向 LSTM（Bidirectional LSTM，BiLSTM）

通俗：**从左读一遍、从右读一遍**，把两个方向的信息拼起来，上下文更全。
专业：前向 $h^\rightarrow$ + 后向 $h^\leftarrow$ concat；序列标注常用。
用法：见 [04/004 文本分类与 NER](../04-自然语言处理/004-文本分类与命名实体识别.md)。

### 序列标注（Sequence Labeling）

通俗：给序列**每个位置贴一个标签**（如每个词是人名/地名/其他），不是整句一个标签。
专业：NER、词性标注、分词等；与文本分类任务范式不同。
用法：见 [04/004](../04-自然语言处理/004-文本分类与命名实体识别.md)。

### BIO 标注（Begin-Inside-Outside，BIO）

通俗：实体**开头标 B、中间标 I、非实体标 O**，解决多字实体边界问题。
专业：BIO/BIOES 序列标注方案；CRF 常作解码层。
用法：见 [04/004](../04-自然语言处理/004-文本分类与命名实体识别.md)。

### 条件随机场（Conditional Random Field，CRF）

通俗：在 BiLSTM 输出上再加一层**全局最优标签序列**解码，避免非法标签组合（如 I 不能开头）。
专业：结构化预测；学习标签转移分数；NER 经典搭配。
用法：见 [04/004](../04-自然语言处理/004-文本分类与命名实体识别.md)。

### BLEU（Bilingual Evaluation Understudy，BLEU）

通俗：机器译文与参考译文** n-gram 重叠多少**；越高越像人工翻译（有局限）。
专业：精确率型指标 + 长度惩罚；机器翻译主流自动指标之一。
用法：见 [04/005 文本生成](../04-自然语言处理/005-文本生成基础.md)。

### ROUGE（Recall-Oriented Understudy for Gisting Evaluation，ROUGE）

通俗：看生成摘要**覆盖了多少参考摘要里的词**；偏召回，摘要任务常用。
专业：ROUGE-N、ROUGE-L 等变体；与 BLEU 侧重不同。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)。

### 困惑度（Perplexity，PPL）

通俗：语言模型**有多「困惑」**——越低说明越能预测下一个词，模型越好。
专业：$PPL=\exp(-\frac{1}{N}\sum\log P(w_i))$；LM 固有评估指标。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)。

### Top-k 采样（Top-k Sampling）

通俗：每步只从**概率最高的 k 个词**里随机抽，砍掉长尾怪词。
专业：与 Top-p 常并用；k 小更保守，k 大更多样。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)。

### 贪心解码（Greedy Decoding）

通俗：每步**只选概率最高的一个词**，最快但容易走进死胡同。
专业：Beam=1 的特例；确定性解码 baseline。
用法：见 [04/005](../04-自然语言处理/005-文本生成基础.md)。

## 七、05 · 大语言模型与 Transformer

### 大语言模型（Large Language Model，LLM）

通俗：**读遍海量文本**练出来的「下一个词预测器」，规模大到能对话、写代码、推理。
专业：通常指参数量达亿级以上的预训练语言模型；能力涌现与 scaling 为研究热点。
用法：见 [05/000 总览](../05-大语言模型与Transformer/000-分类总览与知识图谱.md)。

### Transformer

通俗：全靠**注意力**读序列，不用 RNN 那种一步步排队；现代 LLM 的骨架。
专业：Encoder-Decoder 或 Decoder-only；自注意力 + FFN + 残差 + LayerNorm。
用法：见 [05/002 Transformer 架构](../05-大语言模型与Transformer/002-Transformer架构.md)。

### 自注意力（Self-Attention）

通俗：序列里**每个词都问一遍「谁和我最相关」**，自动加权汇总上下文。
专业：$\mathrm{Attention}(Q,K,V)=\mathrm{softmax}(QK^\top/\sqrt{d_k})V$；Q/K/V 由输入线性投影。
用法：见 [05/001 注意力机制](../05-大语言模型与Transformer/001-注意力机制与自注意力.md)。

### 多头注意力（Multi-Head Attention）

通俗：多组注意力**并行看不同角度**（语法、指代、语义），再拼起来。
专业：$h$ 个头独立投影后 concat 再线性变换。
用法：见 [05/001](../05-大语言模型与Transformer/001-注意力机制与自注意力.md)。

### 位置编码（Positional Encoding）

通俗：告诉模型**词在句子里的顺序**；Transformer 本身不感知顺序。
专业：正弦位置编码或可学习嵌入；RoPE 为 LLM 常用旋转位置编码。
用法：见 [05/002 Transformer](../05-大语言模型与Transformer/002-Transformer架构.md)。

### 预训练（Pre-training）

通俗：用海量文本**自学**语言规律，不需要人工标答案（自己给自己出题）。
专业：自监督目标：下一词预测（GPT）或掩码预测（BERT）。
用法：见 [05/003 预训练与微调](../05-大语言模型与Transformer/003-预训练与微调.md)。

### 微调（Fine-tuning）

通俗：在预训练底子上，用**少量专业数据**再培训，让它胜任具体任务。
专业：继续优化部分或全部参数；SFT 为指令微调常见形式。
用法：见 [05/003](../05-大语言模型与Transformer/003-预训练与微调.md)、[10/005 LoRA](../10-生成式AI/005-LoRA低秩微调.md)。

### 监督微调（Supervised Fine-Tuning，SFT）

通俗：用「问题 → 标准回答」配对数据，教模型**按指令作答**而不是只会续写。
专业：在对话/指令数据集上最小化交叉熵；Chat 模型关键步骤。
用法：见 [05/003](../05-大语言模型与Transformer/003-预训练与微调.md)。

### 参数高效微调（Parameter-Efficient Fine-Tuning，PEFT）

通俗：**只动很少参数**就能适配新任务，省显存、省存储。
专业：LoRA、Adapter、Prefix Tuning 等；可训练参数常 <1%。
用法：见 [05/003](../05-大语言模型与Transformer/003-预训练与微调.md)、[10/005 LoRA](../10-生成式AI/005-LoRA低秩微调.md)。

### 检索增强生成（Retrieval-Augmented Generation，RAG）

通俗：像**开卷考试**——先查资料库，再让模型根据查到的内容回答，减少瞎编。
专业：检索器 + 生成器；向量检索 Top-K chunk 拼入 prompt。
用法：见 [05/004 RAG](../05-大语言模型与Transformer/004-检索增强生成RAG.md)、[09/003 工具调用](../09-提示工程与Agent开发/003-工具调用与函数调用.md)。

### 幻觉（Hallucination）

通俗：模型**一本正经胡说**——编造不存在的事实、文献或数据。
专业：生成式模型固有风险；RAG、工具调用、对齐可缓解但难消除。
用法：见 [05/004 RAG](../05-大语言模型与Transformer/004-检索增强生成RAG.md)、[09/004 Agent 评估](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

### 人类反馈强化学习（Reinforcement Learning from Human Feedback，RLHF）

通俗：让人给多个回答**打分/排序**，训练奖励模型，再强化学习让模型**更讨人喜欢、更安全**。
专业：SFT → RM → PPO/DPO 等；对齐「有用、诚实、无害」。
用法：见 [05/005 对齐与 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)、[07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)。

### 近端策略优化（Proximal Policy Optimization，PPO）

通俗：强化学习里**稳着更新策略**，别一步走太猛把模型练崩；RLHF 经典算法。
专业：带 clip 的策略梯度；与 TRPO 同属 trust region 思想。
用法：见 [05/005 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)、[07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)。

### 直接偏好优化（Direct Preference Optimization，DPO）

通俗：不用单独奖励模型，**直接用人类偏好数据**优化策略，比 PPO 流程简单。
专业：将 RLHF 目标重参数化为偏好对比损失。
用法：见 [05/005 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)。

### KV 缓存（KV Cache）

通俗：生成时**已经算过的 Key/Value 不重复算**，大幅加速自回归推理。
专业：缓存每层过去 token 的 K/V；显存随序列长度增长。
用法：见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 查询 / 键 / 值（Query / Key / Value，Q / K / V）

通俗：注意力里的三个角色——**Query 问「我要找什么」**，Key 答「我有什么特征」，Value 是**真正被加权汇总的内容**。
专业：$\mathrm{Attention}(Q,K,V)=\mathrm{softmax}(QK^\top/\sqrt{d_k})V$；均由输入线性投影。
用法：见 [05/001 注意力机制](../05-大语言模型与Transformer/001-注意力机制与自注意力.md)。

### 前馈网络（Feed-Forward Network，FFN）

通俗：Transformer 层里注意力之后的**两层全连接 MLP**，对每个位置独立做非线性变换。
专业：常 $d\to 4d\to d$；与自注意力并列构成 Transformer Block。
用法：见 [05/002 Transformer 架构](../05-大语言模型与Transformer/002-Transformer架构.md)。

### 因果掩码（Causal Mask）

通俗：生成时**不能偷看后面的词**，注意力矩阵上三角遮掉，保证自回归合法。
专业：Decoder-only LM 必备；与 Encoder 双向注意力相对。
用法：见 [05/002](../05-大语言模型与Transformer/002-Transformer架构.md)。

### 交叉注意力（Cross-Attention）

通俗：Query 来自一边（如文本），Key/Value 来自另一边（如图像 latent）——**跨模态对齐**。
专业：Encoder-Decoder、Stable Diffusion 文本条件注入核心。
用法：见 [05/002 Transformer](../05-大语言模型与Transformer/002-Transformer架构.md)、[10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)。

### 旋转位置编码（Rotary Position Embedding，RoPE）

通俗：把位置信息**编进 Q/K 的旋转角度**，相对距离自然体现；现代 LLM 主流位置编码。
专业：复数域旋转；外推与长上下文扩展为研究热点。
用法：见 [05/002 Transformer](../05-大语言模型与Transformer/002-Transformer架构.md)。

### 生成式预训练 Transformer（Generative Pre-trained Transformer，GPT）

通俗：OpenAI 命名的**只解码、从左到右接龙**的预训练语言模型系列；ChatGPT 即其对话版。
专业：Decoder-only Transformer；自回归下一词预测；GPT-2/3/4 等为不同规模代际。
用法：见 [05/002](../05-大语言模型与Transformer/002-Transformer架构.md)、[05/003 预训练](../05-大语言模型与Transformer/003-预训练与微调.md)。

### 双向编码器表示（Bidirectional Encoder Representations from Transformers，BERT）

通俗：**同时看左右上下文**填被遮住的词，擅长理解而非长文生成。
专业：Encoder-only；MLM + NSP 预训练；微调用于分类/NER 等。
用法：见 [05/003 预训练与微调](../05-大语言模型与Transformer/003-预训练与微调.md)、[04/004 NER](../04-自然语言处理/004-文本分类与命名实体识别.md)。

### 掩码语言建模（Masked Language Modeling，MLM）

通俗：随机盖住一些词，让模型**根据上下文猜被盖的词**——BERT 的预训练任务。
专业：双向上下文；与 GPT 自回归 LM 对照。
用法：见 [05/003](../05-大语言模型与Transformer/003-预训练与微调.md)。

### 对齐（Alignment）

通俗：让模型**有用、诚实、无害**，符合人类价值观与使用意图，而不只是「会接龙」。
专业：RLHF、DPO、Constitutional AI 等均为对齐技术路线。
用法：见 [05/005 对齐与 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)。

### 奖励模型（Reward Model，RM）

通俗：学会**给人更喜欢的回答打高分**，供 RLHF 第三步优化策略用。
专业：Bradley-Terry 或 ranking loss 训练；输入 prompt+response 输出标量奖励。
用法：见 [05/005 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)。

### 奖励黑客（Reward Hacking）

通俗：模型**钻奖励函数空子**——表面分高但人类不满意（啰嗦、讨好、回避等）。
专业：Goodhart 定律在 RLHF 中的体现；需多样化人类反馈与约束。
用法：见 [05/005 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)。

### 上下文学习（In-Context Learning，ICL）

通俗：不改权重，只在 prompt 里**塞示例或说明**，模型就能临时学会新任务。
专业：大模型涌现能力；少样本/零样本提示的理论基础。
用法：见 [09/001 提示工程](../09-提示工程与Agent开发/001-提示工程基础.md)、[05/003 预训练](../05-大语言模型与Transformer/003-预训练与微调.md)。

### 文档分块（Document Chunking）

通俗：长文档**切成小段**再建索引，检索时按段召回，避免整篇塞不进上下文。
专业：chunk size/overlap 影响 RAG 召回与精度；需与 embedding 模型匹配。
用法：见 [05/004 RAG](../05-大语言模型与Transformer/004-检索增强生成RAG.md)。

### 重排序（Reranking）

通俗：向量检索先粗召回 Top-K，再用**更精的模型重新打分排序**，提高最终进 prompt 的质量。
专业：两阶段检索；cross-encoder reranker 常用。
用法：见 [05/004 RAG](../05-大语言模型与Transformer/004-检索增强生成RAG.md)。

## 八、06 · 计算机视觉

### 计算机视觉（Computer Vision，CV）

通俗：让机器**从图像/视频里读出含义**——是什么、在哪、哪块像素属于谁。
专业：分类、检测、分割、生成等任务；CNN 与 ViT 为主力 backbone。
用法：见 [06/000 总览](../06-计算机视觉/000-分类总览与知识图谱.md)。

### 卷积（Convolution）

通俗：用小窗口（卷积核）在图上**滑动检测局部花纹**（边缘、纹理）。
专业：局部连接 + 权值共享；输出为特征图。
用法：见 [06/001 图像表示与卷积](../06-计算机视觉/001-图像表示与卷积运算.md)。

### 卷积神经网络（Convolutional Neural Network，CNN）

通俗：多层卷积 + 池化，**从边缘到部件到整体**层层抽象图像。
专业：归纳偏置：局部性、平移等变；LeNet→ResNet 为经典演进。
用法：见 [06/002 CNN](../06-计算机视觉/002-卷积神经网络CNN.md)。

### 残差网络（Residual Network，ResNet）

通俗：加**抄近道**让超深 CNN 可训，ImageNet 里程碑架构。
专业：残差块 $y=F(x)+x$；解决退化问题。
用法：见 [06/003 经典 CNN 与 ResNet](../06-计算机视觉/003-经典CNN架构与残差网络.md)。

### 目标检测（Object Detection）

通俗：不只说「有什么」，还要框出**在图里哪**；可有多目标。
专业：输出边界框 + 类别；两阶段（R-CNN 系）vs 单阶段（YOLO 系）。
用法：见 [06/004 目标检测与分割](../06-计算机视觉/004-目标检测与图像分割.md)。

### 交并比（Intersection over Union，IoU）

通俗：预测框与真框**重叠多少**；重叠越大定位越准。
专业：$\mathrm{IoU} = |A\cap B|/|A\cup B|$；检测核心指标基础。
用法：见 [06/004](../06-计算机视觉/004-目标检测与图像分割.md)。

### 平均精度均值（mean Average Precision，mAP）

通俗：检测模型「**找得准不准、找得全不全**」的综合分。
专业：多类别 AP 的平均；COCO 等基准常用。
用法：见 [06/004](../06-计算机视觉/004-目标检测与图像分割.md)。

### 视觉 Transformer（Vision Transformer，ViT）

通俗：把图**切成小块当「视觉单词」**，用 Transformer 做分类。
专业：Patch embedding + 位置编码 + Transformer Encoder；大数据预训练下可与 CNN 竞争。
用法：见 [06/005 ViT](../06-计算机视觉/005-视觉Transformer ViT.md)。

### 图像块（Patch）

通俗：把一张图切成 $16\times16$ 等小方块，每块变成一个 token。
专业：ViT 输入单元；$N=(H/P)(W/P)$ 为序列长度。
用法：见 [06/005 ViT](../06-计算机视觉/005-视觉Transformer ViT.md)、[10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)（latent patch）。

### 池化（Pooling）

通俗：把特征图**局部区域压成一个数**（如取最大/平均），缩小尺寸、增强平移鲁棒性。
专业：MaxPool / AvgPool；CNN 降采样与感受野扩大手段。
用法：见 [06/002 CNN](../06-计算机视觉/002-卷积神经网络CNN.md)。

### 感受野（Receptive Field）

通俗：输出特征图上**一个点能「看到」输入图像多大范围**；层越深感受野越大。
专业：由卷积核、步长、层数决定；检测/分割设计需考虑。
用法：见 [06/001 卷积运算](../06-计算机视觉/001-图像表示与卷积运算.md)、[06/002 CNN](../06-计算机视觉/002-卷积神经网络CNN.md)。

### 归纳偏置（Inductive Bias）

通俗：模型结构里**内置的「先验假设」**——CNN 假设局部相关、平移不变；ViT 假设全局关系。
专业：影响小数据泛化与样本效率；CNN vs ViT 对比核心。
用法：见 [06/001](../06-计算机视觉/001-图像表示与卷积运算.md)、[06/005 ViT](../06-计算机视觉/005-视觉Transformer ViT.md)。

### 图像分类（Image Classification）

通俗：给一张图，输出**属于哪一类**（猫/狗/车）；整张图一个标签。
专业：CV 基础任务；与检测（框+类）、分割（像素类）相对。
用法：见 [06/004 检测与分割](../06-计算机视觉/004-目标检测与图像分割.md)、[06/000 总览](../06-计算机视觉/000-分类总览与知识图谱.md)。

### 语义分割（Semantic Segmentation）

通俗：给**每个像素**贴类别标签（路/人/车），不区分同类不同个体。
专业：FCN、U-Net 等；像素级密集预测。
用法：见 [06/004](../06-计算机视觉/004-目标检测与图像分割.md)。

### 实例分割（Instance Segmentation）

通俗：既要**像素级分类**，又要**区分同一类的不同个体**（两辆车各自一块）。
专业：Mask R-CNN 为代表；检测 + 掩码分支。
用法：见 [06/004](../06-计算机视觉/004-目标检测与图像分割.md)。

### 你只看一次（You Only Look Once，YOLO）

通俗：**单阶段检测**——一次前向同时出框和类别，速度极快，适合实时场景。
专业：YOLOv5/v8 等为工业常用；与两阶段 Faster R-CNN 精度-速度权衡不同。
用法：见 [06/004 目标检测](../06-计算机视觉/004-目标检测与图像分割.md)。

### 更快区域卷积神经网络（Faster Region-based Convolutional Neural Network，Faster R-CNN）

通俗：**两阶段检测**——先提候选框，再分类精修；精度高、速度相对慢。
专业：RPN + RoI Pooling；两阶段检测经典 baseline。
用法：见 [06/004](../06-计算机视觉/004-目标检测与图像分割.md)。

### U-Net

通俗：**编码器压尺寸、解码器还原**，中间跳跃连接保留细节——分割与扩散 denoiser 标配结构。
专业：对称 U 形；skip connection 融合多尺度特征。
用法：见 [06/004 分割](../06-计算机视觉/004-目标检测与图像分割.md)、[10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)、[10/004 ControlNet](../10-生成式AI/004-ControlNet与可控生成.md)。

## 九、07 · 强化学习

### 强化学习（Reinforcement Learning，RL）

通俗：智能体在环境里**试错**，根据奖励信号学怎么行动；像训狗做对了给零食。
专业：序列决策；最大化期望累积回报；与监督学习「静态标签」不同。
用法：见 [07/000 总览](../07-强化学习/000-分类总览与知识图谱.md)。

### 马尔可夫决策过程（Markov Decision Process，MDP）

通俗：把「决策问题」形式化成**状态、动作、奖励、转移**四元组；未来只依赖当前状态。
专业：$(S,A,P,R,\gamma)$；马尔可夫性 $P(s_{t+1}|s_t,a_t)$。
用法：见 [07/001 MDP](../07-强化学习/001-马尔可夫决策过程MDP.md)。

### 策略（Policy，$\pi$）

通俗：在某种状态下**选什么动作**的规则；可以是确定的或随机的。
专业：$\pi(a|s)$ 或 $\pi(s)$；策略梯度直接优化 $\pi$。
用法：见 [07/001 MDP](../07-强化学习/001-马尔可夫决策过程MDP.md)、[07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)。

### 价值函数（Value Function，$V$ / $Q$）

通俗：$V$：**这个局面有多好**；$Q$：**在这个局面做这个动作有多好**。
专业：贝尔曼方程递推；$Q$ 可用于 greedy 或 $\epsilon$-greedy。
用法：见 [07/002 价值函数与贝尔曼方程](../07-强化学习/002-价值函数与贝尔曼方程.md)。

### Q 学习（Q-learning）

通俗：不用环境模型，**表格或网络存 Q 值**，往高回报方向更新。
专业：离策略 TD 控制；$Q(s,a)\leftarrow Q + \alpha[r+\gamma\max Q' - Q]$。
用法：见 [07/003 Q 学习](../07-强化学习/003-Q学习与时序差分.md)。

### 策略梯度（Policy Gradient）

通俗：**直接调整策略参数**，让高回报动作概率变大；适合连续动作。
专业：$\nabla J \propto \mathbb{E}[\nabla\log\pi(a|s)\,A]$；REINFORCE、PPO 等。
用法：见 [07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)。

### 深度 Q 网络（Deep Q-Network，DQN）

通俗：用神经网络**近似 Q 函数**，加经验回放和目标网络稳定训练。
专业：Atari 突破；离散动作 RL 经典。
用法：见 [07/003 Q 学习](../07-强化学习/003-Q学习与时序差分.md)。

### 贝尔曼方程（Bellman Equation）

通俗：当前状态价值 = **即时奖励 + 折扣后的下一状态价值**；把长期回报拆成递推。
专业：$V(s)=\mathbb{E}[R+\gamma V(s')]$；动态规划与 RL 核心理论工具。
用法：见 [07/002 价值函数与贝尔曼方程](../07-强化学习/002-价值函数与贝尔曼方程.md)。

### 折扣因子（Discount Factor，$\gamma$）

通俗：**未来奖励打几折**；$\gamma$ 接近 1 更看长远，接近 0 更贪眼前。
专业：MDP 五元组之一；保证无限 horizon 回报收敛（$\gamma<1$）。
用法：见 [07/001 MDP](../07-强化学习/001-马尔可夫决策过程MDP.md)。

### 时序差分学习（Temporal Difference Learning，TD）

通俗：不用等一局结束，**每走一步就用 bootstrap 更新**——用当前估计改进当前估计。
专业：TD(0)、SARSA、Q-learning 基础；结合 MC 与 DP 优点。
用法：见 [07/003 Q 学习与时序差分](../07-强化学习/003-Q学习与时序差分.md)。

### 探索与利用（Exploration-Exploitation Tradeoff）

通俗：**试新路（探索）还是走老路（利用）**——RL 根本困境；只利用可能错过更好策略。
专业：$\epsilon$-greedy、UCB、Thompson sampling 等平衡策略。
用法：见 [07/003 Q 学习](../07-强化学习/003-Q学习与时序差分.md)。

### ε-贪婪（ε-greedy）

通俗：大多数时候选**当前认为最好的动作**，以小概率 $\epsilon$ **随机乱试**探路。
专业：简单探索策略；$\epsilon$ 可随训练衰减。
用法：见 [07/003 Q 学习](../07-强化学习/003-Q学习与时序差分.md)。

### 经验回放（Experience Replay）

通俗：把玩过的 $(s,a,r,s')$ **存进缓冲区随机抽样**训练，打破数据相关性、复用旧经验。
专业：DQN 稳定训练关键；可 prioritized replay 加权重要样本。
用法：见 [07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)。

### 目标网络（Target Network）

通俗：DQN 里**单独一份慢更新的 Q 网络**算 TD 目标，避免自己追自己导致震荡。
专业：周期性硬更新或软更新 $\theta^-\leftarrow\tau\theta+(1-\tau)\theta^-$。
用法：见 [07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)。

### Actor-Critic

通俗：**Actor 选动作，Critic 打分**——一个管策略一个管价值，比纯策略梯度方差小。
专业：A2C/A3C、PPO、SAC 等基础架构；RLHF 中 PPO 亦属此类思想。
用法：见 [07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)、[05/005 RLHF](../05-大语言模型与Transformer/005-对齐与RLHF.md)。

### 优势函数（Advantage Function，$A$）

通俗：**这个动作比平均水平好多少**——$A(s,a)=Q(s,a)-V(s)$；指导策略往「超常发挥」方向更新。
专业：GAE 等估计方法；PPO/Actor-Critic 核心量。
用法：见 [07/004 策略梯度](../07-强化学习/004-策略梯度与深度强化学习.md)。

## 十、08 · AI 工程与 MLOps

### MLOps（Machine Learning Operations，MLOps）

通俗：把模型从「实验室练出来」到**稳定上线、持续监控、坏了再练**的一整套工程实践。
专业：数据 → 训练 → 注册 → 部署 → 监控 → 再训练闭环。
用法：见 [08/000 总览](../08-AI工程与MLOps/000-分类总览与知识图谱.md)。

### 数据管线（Data Pipeline）

通俗：数据从采集、清洗、特征加工到**稳定喂给模型**的流水线；Garbage In, Garbage Out。
专业：ETL/ELT、特征库、训练-服务一致性。
用法：见 [08/001 数据管线与特征工程](../08-AI工程与MLOps/001-数据管线与特征工程.md)。

### 可复现性（Reproducibility）

通俗：**别人用你的代码和数据能复现同样结果**；固定种子、记录版本是基本功。
专业：随机性、数据、代码、环境四要素锁定。
用法：见 [08/002 实验管理与可复现性](../08-AI工程与MLOps/002-实验管理与可复现性.md)。

### 推理（Inference）

通俗：模型**练好之后拿来用**——输入新数据，输出预测；相对「训练」而言。
专业：前向传播；关注延迟、吞吐、成本而非梯度。
用法：见 [08/003 模型部署与推理服务](../08-AI工程与MLOps/003-模型部署与推理服务.md)。

### 延迟与吞吐（Latency & Throughput）

通俗：**延迟**=一次请求等多久；**吞吐**=单位时间处理多少请求；往往要折中。
专业：P50/P95/P99 延迟；QPS、tokens/s。
用法：见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 数据漂移（Data Drift）

通俗：线上来的数据和训练时**不一样了**（用户变了、季节变了），模型悄悄变差。
专业：输入分布 $P(X)$ 变化；PSI、KS 检验等监控。
用法：见 [08/004 监控与模型漂移](../08-AI工程与MLOps/004-监控与模型漂移.md)。

### 概念漂移（Concept Drift）

通俗：**同样输入，正确答案的含义变了**（如「正常交易」定义随政策变）。
专业：$P(Y|X)$ 变化；需重新标注或再训练。
用法：见 [08/004](../08-AI工程与MLOps/004-监控与模型漂移.md)。

### 量化（Quantization）

通俗：把权重从 32 位浮点**压到 16/8 位整数**，模型更小、算更快，略损精度。
专业：PTQ（训后量化）、QAT（量化感知训练）；LLM 常用 GPTQ/AWQ。
用法：见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 知识蒸馏（Knowledge Distillation）

通俗：**大模型教小模型**，小模型模仿大模型输出，体积小、推理快。
专业：软标签 + 硬标签联合训练；DistilBERT 等为范例。
用法：见 [08/005](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 金丝雀发布（Canary Release）

通俗：新版本**先给 5% 流量**，没问题再全量——上线别一把梭。
专业：灰度/金丝雀是降低部署风险的发布策略。
用法：见 [08/003 模型部署](../08-AI工程与MLOps/003-模型部署与推理服务.md)。

### 特征工程（Feature Engineering）

通俗：把原始数据**加工成模型更好用的特征**——填缺失、编码类别、构造交叉特征等。
专业：表格 ML 核心；深度学习部分被表示学习取代，但业务特征仍关键。
用法：见 [08/001 数据管线与特征工程](../08-AI工程与MLOps/001-数据管线与特征工程.md)。

### 特征库（Feature Store）

通俗：**线上线下共用同一套特征定义与计算**，避免训练时用的特征和上线时算的不一样。
专业：Point-in-time 正确性；Feast、Tecton 等为工业方案。
用法：见 [08/001](../08-AI工程与MLOps/001-数据管线与特征工程.md)。

### 数据泄漏（Data Leakage）

通俗：**测试信息偷偷混进训练**——离线指标虚高，上线必翻车。
专业：未来信息、目标衍生特征、错误划分导致；ML 项目头号坑之一。
用法：见 [08/001](../08-AI工程与MLOps/001-数据管线与特征工程.md)。

### 训练-服务偏差（Training-Serving Skew）

通俗：**线下训练用的特征算法和线上实时算的不一致**，模型线上效果掉崖。
专业：与特征库、版本锁定、一致 pipeline 相对；skew 监控必备。
用法：见 [08/001](../08-AI工程与MLOps/001-数据管线与特征工程.md)。

### 群体稳定性指数（Population Stability Index，PSI）

通俗：比较两批数据**分布差多大**；PSI 超阈值常报警数据漂移。
专业：$\sum (p_i-q_i)\ln(p_i/q_i)$ 类指标；风控与 MLOps 常用。
用法：见 [08/004 监控与模型漂移](../08-AI工程与MLOps/004-监控与模型漂移.md)。

### 蓝绿部署（Blue-Green Deployment）

通俗：**两套环境随时切换**——蓝环境跑旧版、绿环境跑新版，一键切流量，回滚快。
专业：与金丝雀（渐进切流量）互补的发布策略。
用法：见 [08/003 模型部署](../08-AI工程与MLOps/003-模型部署与推理服务.md)。

### A/B 测试（A/B Testing）

通俗：**随机分用户**对比新旧模型真实业务指标（点击率、转化等），用数据决定上不上线。
专业：实验设计 + 统计显著性；与离线指标互补。
用法：见 [08/003 模型部署](../08-AI工程与MLOps/003-模型部署与推理服务.md)。

### 动态批处理（Dynamic Batching）

通俗：推理时**把短时间内的多个请求拼成一批**一起算，提高 GPU 利用率。
专业：等待窗口 + 最大 batch size；CV/传统 ML 推理常用。
用法：见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 连续批处理（Continuous Batching）

通俗：LLM 生成时**不同请求长度不一**，算完短的立刻补新的，GPU 不空转。
专业：Orca/vLLM 核心思想；与 static batching 相对。
用法：见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 分页注意力（PagedAttention）

通俗：把 KV Cache **像内存分页一样切块管理**，减少显存碎片，同卡跑更多并发。
专业：vLLM 核心技术；借鉴 OS 虚拟内存思想。
用法：见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 可变长度高效 LLM 服务（vLLM）

通俗：开源 **LLM 高吞吐推理引擎**，PagedAttention + 连续批处理，工业部署常用。
专业：Serving 框架；与 TGI、TensorRT-LLM 等为同类选型。
用法：见 [08/005 成本与性能优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

## 十一、09 · 提示工程与 Agent 开发

### 提示工程（Prompt Engineering）

通俗：**怎么跟大模型说话**才能让它答得更好——指令、例子、格式都算 prompt 设计。
专业：零样本/少样本、角色设定、输出约束等；无梯度更新，靠上下文引导。
用法：见 [09/001 提示工程基础](../09-提示工程与Agent开发/001-提示工程基础.md)。

### 提示（Prompt）

通俗：你发给模型的**全部输入上下文**（系统说明 + 用户问题 + 检索内容等）。
专业：Decoder-only 模型的条件前缀；token 长度影响成本与效果。
用法：见 [09/001](../09-提示工程与Agent开发/001-提示工程基础.md)。

### 少样本学习（Few-shot）

通俗：在 prompt 里**给几个示范例子**，让模型照猫画虎；不用改权重。
专业：In-Context Learning（ICL）的一种；示例质量与顺序敏感。
用法：见 [09/001](../09-提示工程与Agent开发/001-提示工程基础.md)。

### 思维链（Chain-of-Thought，CoT）

通俗：让模型**一步步写出推理过程**再答，复杂题正确率常明显提升。
专业：「Let's think step by step」可零样本触发；少样本 CoT 在示例中带推理链。
用法：见 [09/002 思维链与推理](../09-提示工程与Agent开发/002-思维链与推理.md)。

### 自洽性（Self-Consistency）

通俗：同一题**采样多条推理路径**，对最终答案**投票**，比单路径更稳。
专业：CoT + 多数表决；成本随采样次数线性增。
用法：见 [09/002](../09-提示工程与Agent开发/002-思维链与推理.md)。

### ReAct（Reason + Act，ReAct）

通俗：**边想边动手**——思考 → 调工具 → 看结果 → 再想；Agent 核心范式之一。
专业：Thought → Action → Observation 循环；结合 LLM 与外部工具。
用法：见 [09/002](../09-提示工程与Agent开发/002-思维链与推理.md)、[09/004 Agent 架构](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

### 工具调用 / 函数调用（Tool Calling / Function Calling）

通俗：让模型**输出结构化指令**去调 API（查天气、算数、查库），而不是瞎编结果。
专业：JSON Schema 描述工具；模型生成 arguments，宿主执行后回填 observation。
用法：见 [09/003 工具调用与函数调用](../09-提示工程与Agent开发/003-工具调用与函数调用.md)。

### 智能体（Agent）

通俗：LLM + **规划 + 记忆 + 工具**，能多步完成目标的「AI 办事员」；区别于单次问答。
专业：Agent Loop：感知 → 规划 → 行动 → 反馈；多 Agent 协作可分工。
用法：见 [09/004 Agent 架构与评估](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

### 模型上下文协议（Model Context Protocol，MCP）

通俗：让 AI 应用**标准化连接外部工具与数据源**的开放协议（类似 USB 口）。
专业：Cursor 等 IDE 预置 MCP 服务；工具/资源统一发现与调用。
用法：见仓库 `AGENTS.md` MCP 节、[09/003 工具调用](../09-提示工程与Agent开发/003-工具调用与函数调用.md)。

### 规划（Planning）

通俗：Agent **把大目标拆成多步小任务**，决定先做什么后做什么。
专业：Agent 四大组件之一；ReAct、Plan-and-Solve 等为实现范式。
用法：见 [09/004 Agent 架构](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

### 记忆（Memory）

通俗：Agent **记住对话历史、用户偏好、中间结果**；短期 vs 长期记忆是 Agent 与单次问答的本质区别。
专业：Buffer、Summary、Vector store 等实现；RAG 可视为外部记忆。
用法：见 [09/004 Agent 架构](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

### 安全护栏（Guardrails）

通俗：**限制 Agent 能做什么、输出什么**——敏感操作要确认、有害内容要拦截。
专业：规则 + 模型双层；生产 Agent 必备。
用法：见 [09/004 Agent 架构](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

### LLM 作裁判（LLM-as-a-Judge）

通俗：用**另一个大模型**给 Agent 回答打分/比较，评估开放式任务质量。
专业：G-Eval、 pairwise comparison；人工评估成本高时的替代。
用法：见 [09/004 Agent 评估](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

### 提示注入（Prompt Injection）

通俗：攻击者通过输入**篡改系统指令**，让模型执行非预期操作（泄密、越权调工具）。
专业：Agent/工具调用核心安全威胁；需权限最小化与输入隔离。
用法：见 [09/003 工具调用](../09-提示工程与Agent开发/003-工具调用与函数调用.md)、[09/004 Agent](../09-提示工程与Agent开发/004-Agent架构与评估.md)。

## 十二、10 · 生成式 AI

### 生成式人工智能（Generative AI，GenAI）

通俗：AI **造出新内容**（文、图、音、视频），不只是分类或预测。
专业：学习数据分布并采样；判别式 vs 生成式为不同建模范式。
用法：见 [10/000 总览](../10-生成式AI/000-分类总览与知识图谱.md)。

### 扩散模型（Diffusion Model）

通俗：**先加噪再加噪**把图变成雪花屏，再教网络**一步步去噪**；生成时从纯噪声倒放。
专业：DDPM 等；学习 $\epsilon_\theta(x_t,t)$ 或 score；Stable Diffusion 为 latent 版。
用法：见 [10/001 扩散模型 Diffusion 基础](../10-生成式AI/001-扩散模型Diffusion基础.md)。

### 去噪扩散概率模型（Denoising Diffusion Probabilistic Models，DDPM）

通俗：扩散模型的经典公式化名称——前向加噪马尔可夫链 + 学习反向去噪。
专业：Ho et al. 2020；与 score-based 模型等价视角存在。
用法：见 [10/001](../10-生成式AI/001-扩散模型Diffusion基础.md)。

### 潜空间扩散（Latent Diffusion）

通俗：不在像素上扩散（太慢），先**压缩到小 latent**，在 latent 里加噪去噪，再解码回图。
专业：Stable Diffusion = VAE + U-Net 扩散 + CLIP 文本条件。
用法：见 [10/001](../10-生成式AI/001-扩散模型Diffusion基础.md)。

### Stable Diffusion（Stable Diffusion，SD）

通俗：开源文生图**工业标准**之一；可本地跑，生态（LoRA、ControlNet）极丰富。
专业：Latent Diffusion Model；SD 1.5/SDXL/SD3 为多代版本。
用法：见 [10/001](../10-生成式AI/001-扩散模型Diffusion基础.md)、[10/005 LoRA](../10-生成式AI/005-LoRA低秩微调.md)。

### 生成对抗网络（Generative Adversarial Network，GAN）

通俗：**造假者 G** 和 **鉴定师 D** 互相较劲，G 最终能造以假乱真的样本。
专业：minimax 博弈；一次前向生成快，但训练不稳定、模式崩塌。
用法：见 [10/002 GAN 基础](../10-生成式AI/002-生成对抗网络GAN基础.md)。

### 模式崩塌（Mode Collapse）

通俗：GAN 只造**几种固定花样**，多样性丧失；像造假者只会画同一幅画。
专业：生成器 collapse 到少数 modes；扩散相对更稳。
用法：见 [10/002 GAN](../10-生成式AI/002-生成对抗网络GAN基础.md)。

### 对比语言-图像预训练（Contrastive Language-Image Pre-training，CLIP）

通俗：用海量图文对训练，让**匹配的图和文在向量空间里靠近**；零样本分类、文生图条件编码。
专业：双塔 + InfoNCE 对比损失；OpenAI CLIP / OpenCLIP 等。
用法：见 [10/003 多模态 CLIP](../10-生成式AI/003-多模态CLIP基础.md)。

### ControlNet

通俗：给 Stable Diffusion 加**结构条件通道**（线稿、深度、姿态），生成听指挥又不破坏原模型。
专业：冻结 SD + 可训练旁路 + 零卷积注入。
用法：见 [10/004 ControlNet 与可控生成](../10-生成式AI/004-ControlNet与可控生成.md)。

### 低秩适配（Low-Rank Adaptation，LoRA）

通俗：冻结大模型，只训一小块**低秩补丁** $BA$ 定制画风/角色/领域；省显存省存储。
专业：$W = W_0 + \frac{\alpha}{r}BA$；LLM 与 SD 均广泛使用。
用法：见 [10/005 LoRA](../10-生成式AI/005-LoRA低秩微调.md)、[05/003 预训练与微调](../05-大语言模型与Transformer/003-预训练与微调.md)。

### 流匹配（Flow Matching）

通俗：学一个**流速场**，让粒子从噪声沿流线漂到数据；常比多步扩散采样更快。
专业：CFM 回归 $v_\theta(x_t,t) \approx u_t$；Rectified Flow、SD3/Flux 等采用。
用法：见 [10/006 Flow Matching 基础](../10-生成式AI/006-Flow-Matching基础.md)。

### 条件流匹配（Conditional Flow Matching，CFM）

通俗：在噪声样本与真实样本之间**架一条路径**（常线性），让网络学「此刻该往哪流」。
专业：$x_t=(1-t)x_0+tx_1$，目标速度 $x_1-x_0$（线性情形）。
用法：见 [10/006](../10-生成式AI/006-Flow-Matching基础.md)。

### 视频生成（Video Generation）

通俗：不只生成一张图，而是**连贯的一串帧**；难点是时间一致性与算力。
专业：T2V / I2V；Temporal Attention、SVD 等；见 [10/007 视频生成](../10-生成式AI/007-视频生成基础.md)。

### 图生视频（Image-to-Video，I2V）

通俗：**第一帧锁死**，只生成后面怎么动；Stable Video Diffusion 为代表。
专业：条件含首帧 latent + motion 强度；比 T2V 结构更可控。
用法：见 [10/007 视频生成基础](../10-生成式AI/007-视频生成基础.md)。

### 多 LoRA 合并（Multi-LoRA Merge）

通俗：多个 LoRA **按权重叠加**到同一底模；ComfyUI 里链式加载或离线 merge 进 checkpoint。
专业：$W_{\mathrm{eff}}=W_0+\sum_i w_i \frac{\alpha_i}{r_i}B_iA_i$；注意风格冲突。
用法：见 [10/008 多 LoRA 合并与推理实践](../10-生成式AI/008-多LoRA合并与推理实践.md)。

### 变分自编码器（Variational Autoencoder，VAE）

通俗：把图像**压到 latent 小空间再还原**；latent 里每点对应一张「像真图」的样本，Stable Diffusion 靠它压缩。
专业：Encoder 输出 $\mu,\sigma$，重参数化采样；ELBO 训练；与 GAN、扩散并列为生成范式。
用法：见 [10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)、[10/002 GAN](../10-生成式AI/002-生成对抗网络GAN基础.md)。

### 无分类器引导（Classifier-Free Guidance，CFG）

通俗：生成时**同时跑「有条件」和「无条件」**，放大两者差距，让 prompt 控制更强。
专业：$\hat\epsilon=\epsilon_u+w(\epsilon_c-\epsilon_u)$；$w$ 为 guidance scale；SD 文生图关键。
用法：见 [10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)、[10/006 Flow Matching](../10-生成式AI/006-Flow-Matching基础.md)。

### 噪声日程（Noise Schedule）

通俗：扩散前向过程**每一步加多少噪**的预设曲线；影响训练稳定与采样质量。
专业：$\beta_t$ 或 $\bar\alpha_t$ 序列；linear/cosine schedule 等。
用法：见 [10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)。

### 去噪扩散隐式模型（Denoising Diffusion Implicit Models，DDIM）

通俗：**少步跳跃采样**的扩散推理方法，不必像 DDPM 那样一步步走完 1000 步。
专业：非马尔可夫采样；10~50 步可出图；与 SD 常用 sampler 相关。
用法：见 [10/001 扩散](../10-生成式AI/001-扩散模型Diffusion基础.md)。

### 对比学习（Contrastive Learning）

通俗：**拉近正样本对、推远负样本对**——CLIP 图文对齐、SimCLR 图像表征均用此思路。
专业：InfoNCE 损失；batch 内负采样。
用法：见 [10/003 CLIP](../10-生成式AI/003-多模态CLIP基础.md)。

### 文本到视频（Text-to-Video，T2V）

通俗：**给文字描述，生成整段视频**；比文生图多时间维，一致性更难。
专业：Sora、Runway 等为代表；Temporal Attention 保帧间连贯。
用法：见 [10/007 视频生成基础](../10-生成式AI/007-视频生成基础.md)（与 I2V 词条对照）。

### 时序注意力（Temporal Attention）

通俗：视频模型里**让不同帧之间互相关注**，保持动作、外观前后一致。
专业：沿时间维 self-attention 或 3D conv；T2V/I2V 核心组件。
用法：见 [10/007 视频生成](../10-生成式AI/007-视频生成基础.md)。

## 十三、跨领域通用

### 人工智能（Artificial Intelligence，AI）

通俗：让机器表现出**需要智能**的行为（识别、推理、生成、决策等）的总称。
专业：本库聚焦**机器学习 / 深度学习**路线下的 AI 开发实践。
用法：见 [00/000 总览](./000-分类总览与知识图谱.md)。

### 机器学习（Machine Learning，ML）

通俗：让程序**从数据里自动学规律**，而不是人写死 if-else。
专业：统计学习 + 优化；监督/无监督/强化等为子范式。
用法：见 [02/000 总览](../02-机器学习基础/000-分类总览与知识图谱.md)。

### 深度学习（Deep Learning，DL）

通俗：用**很多层的神经网络**自动学特征；图像、语言、语音的主力方法。
专业：表示学习；依赖 GPU 与大规模数据。
用法：见 [03/000 总览](../03-深度学习基础/000-分类总览与知识图谱.md)。

### 特征（Feature）

通俗：描述一条样本的**可测量属性**（年龄、像素、词频等）。
专业：输入向量 $\mathbf{x}$ 的各维；特征工程 vs 表示学习。
用法：见 [02/002 线性回归](../02-机器学习基础/002-线性回归与逻辑回归.md)、[08/001 特征工程](../08-AI工程与MLOps/001-数据管线与特征工程.md)。

### 标签（Label）

通俗：训练数据里的**标准答案**（类别、数值、框坐标等）。
专业：监督学习的 $y$；测试集标签仅用于评估不可用于调参。
用法：见 [02/001 监督学习](../02-机器学习基础/001-监督学习与无监督学习.md)。

### 超参数（Hyperparameter）

通俗：人**训练前设定**的旋钮（学习率、层数、LoRA 秩 $r$），不是模型自己学出来的权重。
专业：区别于参数 $\theta$；常用验证集选择。
用法：见 [02/003 模型评估](../02-机器学习基础/003-模型评估与指标.md)、[03/003 优化器](../03-深度学习基础/003-优化器与学习率调度.md)。

### 训练集 / 验证集 / 测试集（Train / Validation / Test Set）

通俗：练习册 / 模拟考 / **最终一次大考**；测试集只能最后用一次，不能偷看调参。
专业：Hold-out 划分；K 折为验证的变体。
用法：见 [02/003 模型评估](../02-机器学习基础/003-模型评估与指标.md)。

### 嵌入（Embedding）

通俗：把离散对象（词、用户、图像块）变成**固定长度的数字向量**，方便计算相似度或喂给网络。
专业：可学习查表或编码器输出；RAG 检索依赖文本 embedding。
用法：见 [04/001 词向量](../04-自然语言处理/001-文本表示与词向量.md)、[05/004 RAG](../05-大语言模型与Transformer/004-检索增强生成RAG.md)。

### 嵌入向量 / 向量数据库（Embedding & Vector Database）

通俗：语义相近的文本**向量距离近**；向量库专门存向量、做**近似最近邻**快搜。
专业：HNSW、IVF 等 ANN 索引；RAG 检索后端。
用法：见 [05/004 RAG](../05-大语言模型与Transformer/004-检索增强生成RAG.md)。

### 批量（Batch）

通俗：一次喂给模型**多条样本**一起算；太大占显存，太小不稳定。
专业：mini-batch 为 SGD 默认；推理侧 dynamic batching 提吞吐。
用法：见 [03/002 反向传播](../03-深度学习基础/002-反向传播与梯度下降.md)、[08/005 成本优化](../08-AI工程与MLOps/005-成本与性能优化.md)。

### 检查点（Checkpoint）

通俗：训练过程中**存下来的模型快照**，可恢复训练或部署推理。
专业：含权重、优化器状态（可选）；LoRA 可只存小补丁文件。
用法：见 [08/002 实验管理](../08-AI工程与MLOps/002-实验管理与可复现性.md)、[10/008 多 LoRA](../10-生成式AI/008-多LoRA合并与推理实践.md)。

### 零样本（Zero-shot）

通俗：**没专门训过这个任务**，靠预训练通用能力 + prompt 直接做。
专业：CLIP 零样本分类、LLM 零样本 CoT 等；与 few-shot 相对。
用法：见 [09/001 提示工程](../09-提示工程与Agent开发/001-提示工程基础.md)、[10/003 CLIP](../10-生成式AI/003-多模态CLIP基础.md)。
