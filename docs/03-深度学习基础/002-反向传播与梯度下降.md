# 002 · 反向传播与梯度下降

> 本文用大白话回答：网络里那么多权重，是怎么被"学"出来的？"反向传播"和"梯度下降"听起来很唬人，它俩到底各干什么活？
>
> 读完你会知道：这俩是黄金搭档——一个负责"算出该怎么调"，一个负责"照着调"。

## 一、一句话先说清

- **梯度下降**：**照着"最快下坡的方向"，一小步一小步地把参数往"更好"的方向挪。**
- **反向传播**：**一种高效算出"每个参数该往哪挪、挪多少"的方法。**

配合起来就是：**反向传播负责"算账"（算出每个参数的调整方向），梯度下降负责"花钱"（照着账单更新参数）。** 训练网络就是这俩不停轮流干活。

## 二、打个比方：蒙眼下山

把"训练网络"想象成**蒙着眼睛下山**，目标是走到山谷最低处：

| 下山场景 | 训练网络里的对应 | 含义 |
| --- | --- | --- |
| 你所在的海拔高度 | **损失函数**（衡量预测有多差） | 越低越好 |
| 脚下最陡的上坡方向 | **梯度** | 它的反方向就是"最快下坡" |
| 朝下坡方向迈一小步 | **梯度下降** | 每步把参数挪一点 |
| 怎么快速搞清"这一步往哪迈" | **反向传播** | 不用一个个参数瞎试，一次回溯算清楚 |

- **梯度（gradient）**：可以理解为"当前脚下最陡的上坡方向"。既然要下山（让损失变小），就朝它的**反方向**走。
- **反向传播（backpropagation）**：如果每个参数都单独去试"我改一点损失会怎样"，成本高得离谱。反向传播用一个巧妙的数学技巧（链式法则），**从山顶（输出误差）一路往回倒推到每个参数**，一次就把所有参数的"该往哪调"算出来。

## 三、它到底解决什么问题

### 问题 1：怎么知道"当前有多差"——损失函数

得先有个分数衡量"预测差多少"。以回归任务的均方误差为例，单个样本：

$$
L = \frac{1}{2}(\hat{y} - y)^2
$$

> 对齐：$\hat{y}$ 是网络的预测，$y$ 是真实答案，差得越多 $L$ 越大。这就是前面说的"海拔高度"。

对所有样本取平均就是总目标 $J(\theta) = \frac{1}{m}\sum_{i=1}^{m} L^{(i)}$（$\theta$ 指网络里全部权重和偏置）。

### 问题 2：知道差多少后，往哪调——梯度下降

参数朝"负梯度方向"（最快下坡）挪一步：

$$
\theta \leftarrow \theta - \eta \, \nabla_\theta J(\theta)
$$

> 对齐：$\nabla_\theta J$ 就是"最陡上坡方向"，前面加负号就是"往下坡走"；$\eta$（**学习率**）是"步子迈多大"。

步子大小很讲究：

| 学习率 | 后果 | 类比 |
| --- | --- | --- |
| 太大 | 震荡甚至发散 | 一步迈过山谷，跳到对面更高处 |
| 太小 | 收敛极慢 | 一次挪一厘米，下山下到天黑 |

每次用多少数据来算这一步，又分三种：

- **批量梯度下降（BGD）**：用全部样本算一次梯度，稳但慢。
- **随机梯度下降（SGD）**：每次只用一个样本，快但方向抖。
- **小批量（Mini-batch）**：一次用一小批，折中，工业界主流。

### 问题 3：几百万个参数，梯度怎么高效算——反向传播

假设某层 $z = \mathbf{w}^\top \mathbf{a} + b$，经激活 $out = \sigma(z)$，最终得到损失 $L$。想知道"某个权重 $w_i$ 变一点，$L$ 会变多少"，用**链式法则**（"A 影响 B、B 影响 C，则 A 对 C 的影响 = 两段影响相乘"）：

$$
\frac{\partial L}{\partial w_i}
= \frac{\partial L}{\partial out}\cdot\frac{\partial out}{\partial z}\cdot\frac{\partial z}{\partial w_i}
= \underbrace{\frac{\partial L}{\partial out}\,\sigma'(z)}_{\text{记为误差项 }\delta}\cdot a_i
$$

> 对齐：这一长串就是"损失 ← 输出 ← 加权和 ← 权重"这条影响链上，每一段影响相乘。

关键妙处：**上一层的误差项 $\delta$ 可以由下一层的 $\delta$ 直接递推出来**。所以只要从输出层向输入层"倒着传一遍"，就能一次算出所有参数的梯度，计算量和"正着算一遍预测"差不多——这就是它高效的根本原因，也是它叫"反向传播"的由来。

## 四、案例解析：手算一次参数更新

用最简单的单神经元（不加激活，直接输出）：$\hat{y} = w x + b$，损失 $L=\frac{1}{2}(\hat{y}-y)^2$。

给定一个样本 $x=2,\ y=4$，初始 $w=0,\ b=0$，学习率 $\eta=0.1$。走一遍完整流程：

**第 1 步·前向（先算预测和损失）：** $\hat{y} = 0\times 2 + 0 = 0$，$L = \frac{1}{2}(0-4)^2 = 8$。

**第 2 步·反向（算梯度，即"该往哪调"）：**

$$
\frac{\partial L}{\partial \hat{y}} = \hat{y}-y = -4,\quad
\frac{\partial L}{\partial w} = (\hat{y}-y)\,x = -8,\quad
\frac{\partial L}{\partial b} = (\hat{y}-y) = -4
$$

**第 3 步·更新（照着账单调参数）：**

$$
w \leftarrow 0 - 0.1\times(-8) = 0.8,\qquad
b \leftarrow 0 - 0.1\times(-4) = 0.4
$$

**第 4 步·验证是否变好：** 新预测 $\hat{y}=0.8\times2+0.4 = 2.0$，新损失 $L=\frac{1}{2}(2.0-4)^2 = 2.0$，比原来的 $8$ 明显小了。说明这一步确实让网络变好了；如此反复迭代，$\hat{y}$ 会越来越接近真实值 $4$。

> 这就是"**前向算损失 → 反向算梯度 → 更新参数 → 损失下降**"的完整闭环，也是所有深度学习训练的核心节奏。

## 五、常见误区与边界

- **误区："反向传播是一种训练算法"**：不是。反向传播只负责**高效算梯度**；真正"更新参数"的是梯度下降及其升级版（见 [003 · 优化器与学习率调度](./003-优化器与学习率调度.md)）。
- **误区："学习率随便设一个就行"**：学习率是最关键的超参数之一，通常要精心调整或用调度策略（预热、退火）。
- **边界·梯度消失/爆炸**：网络很深时，梯度在层层相乘中可能趋近 0（消失）或变得极大（爆炸），需要 ReLU、残差连接、批归一化、梯度裁剪等手段缓解。

## 六、一句话总结

- **反向传播算账（每个参数该怎么调），梯度下降花钱（照着调）**，两者轮流迭代把网络练好。
- 损失函数衡量"有多差"，梯度指出"下坡方向"，学习率决定"步子多大"。
- 手算案例展示了"前向 → 反向 → 更新 → 损失下降"这一训练闭环。
