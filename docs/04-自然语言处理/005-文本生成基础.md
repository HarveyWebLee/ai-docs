# 005 · 文本生成基础

> 本文用大白话回答：机器怎么"一个字一个字往外蹦"写出一段话？文本分类是"读完全文给标签"，生成为什么是"边读边写"？温度、Top-k 这些采样参数又在控制什么？
>
> 读完你会知道：文本生成 = 把语言模型当"超级自动补全"——根据已经写出的内容，反复预测下一个词，拼成完整句子。

## 一、一句话先说清

**文本生成 = 让模型按顺序预测"下一个词/字是什么"，再把预测结果接在后面，循环往复直到写完。**

- **语言模型（Language Model，LM）**：给一段前缀，输出下一个 token 的概率分布。
- **自回归生成（autoregressive generation）**：用自己刚生成的 token 作为下一步输入，像**接龙**一样往后写。

手机输入法的联想、ChatGPT 的回答、代码补全，底层都是这一套思路（大模型版见 [05 · 大语言模型](../05-大语言模型与Transformer/000-分类总览与知识图谱.md)）。

## 二、打个比方：接龙游戏 vs 判作文

| 任务 | 怎么读 | 怎么输出 | 类比 |
| --- | --- | --- | --- |
| **文本分类**（见 [004](./004-文本分类与命名实体识别.md)） | 读完全文 | 给一个总标签 | 老师读完作文打总分 |
| **文本生成** | 读已有前缀 | 每次只吐**下一个**词，再接着写 | 成语接龙：知道"床前明月光"，接下一句 |

生成不是"一次性输出整段"，而是**一步一步接龙**——每步只决定"下一个 token 是谁"。

## 三、它到底解决什么问题

### 问题 1：怎么把"理解语言"变成"写出语言"

[003 RNN/LSTM](./003-循环神经网络RNN与LSTM.md) 能读序列；生成只需把训练目标反过来：**不预测整句标签，而预测"给定前文，下一个 token 是什么"**。

训练目标（最大似然）：

$$
\max \sum_{t=1}^{T} \log P(w_t \mid w_1, \dots, w_{t-1}; \theta)
$$

> 对齐：对每个位置 $t$，让模型给"真正的下一个词 $w_t$"尽量高的概率——就是在练"接龙能力"。

### 问题 2：训练好了，推理时怎么"接龙写下去"

**朴素做法**：每步选概率最大的词（**贪心解码**）——快，但容易走进死胡同、输出重复无聊。

**更好做法**：保留多条候选路径，或用**随机采样**增加多样性。

| 策略 | 大白话 | 特点 |
| --- | --- | --- |
| 贪心（Greedy） | 每步选最可能的那个 | 快、确定性强，易重复 |
| Beam Search（束搜索） | 每步保留 top-B 条接龙路径 | 质量更稳，计算量 ×B |
| 随机采样 | 按概率随机抽下一个词 | 多样，但可能胡言乱语 |

### 问题 3：生成太死板或太胡来——温度与 Top-k / Top-p

- **Temperature（温度）** $T$：把 logits 除以 $T$ 再 softmax。$T<1$ 分布更"尖"（更保守、更确定）；$T>1$ 更"平"（更随机、更有创意）。
- **Top-k**：只从概率最高的 $k$ 个词里抽。
- **Top-p（nucleus）**：从累计概率达到 $p$ 的最小词集合里抽。

> 对齐：温度像"创意旋钮"——拧低更稳，拧高更飘；Top-k/Top-p 像"只在靠谱候选里抽签"，避免抽到极低概率的怪词。

## 四、专业视角（与大白话对齐）

### 4.1 经典架构路线

| 范式 | 结构 | 典型用途 |
| --- | --- | --- |
| **N-gram / 统计语言模型** | 查表数共现频率 | 早期输入法、简单基线 |
| **Seq2Seq + 注意力** | 编码器读输入，解码器逐步生成输出 | 机器翻译、摘要（RNN 时代） |
| **Decoder-only Transformer** | 因果掩码自注意力，只预测下一 token | GPT、现代 LLM（见 [05/002](../05-大语言模型与Transformer/002-Transformer架构.md)） |

Encoder-Decoder（Seq2Seq）适合"给一段、写一段"（翻译）；Decoder-only 适合"给前缀、续写"（对话、写作）。

### 4.2 解码公式

设第 $t$ 步 logits 为 $\mathbf{z}_t$，则：

$$
P(w \mid \text{context}) = \text{softmax}(\mathbf{z}_t / T)
$$

Beam Search 维护 $B$ 条部分序列，每步扩展并保留累计 log 概率最高的 $B$ 条。

### 4.3 评价生成质量

- **BLEU / ROUGE**：与参考译文/摘要比 n-gram 重叠（翻译、摘要常用）；
- **困惑度（Perplexity）**：语言模型固有指标，越低越好；
- **人工评估**：流畅度、事实性、有用性——开放域对话往往离不开人评。

## 五、案例解析：字符级 bigram 接龙

用极简统计语言模型演示"接龙"逻辑。训练语料只有一句：`"我爱中国"`（字符级）。

统计相邻字共现：

- `我` 后面 → `爱`（100%）
- `爱` 后面 → `中`（100%）
- `中` 后面 → `国`（100%）

给定前缀 `"我爱"`，查表得下一字必为 `"中"`；再接上得 `"我爱中"`，下一步 `"国"`，完成 `"我爱中国"`。

这就是**最朴素的语言模型 + 贪心解码**。真实系统用神经网络输出整个词表上的概率（见 [05/003 预训练与微调](../05-大语言模型与Transformer/003-预训练与微调.md)），但"查条件概率 → 选下一个 token → 拼接"的循环不变。

## 六、常见误区与边界

- **误区："生成 = 搜索数据库里的句子"**：现代 LM 是**组合泛化**，能造训练集里没出现过的句子，也可能**幻觉**（胡说看似合理的事实）。
- **误区："温度越高越好"**：过高导致乱码；生产环境常偏低温度 + Top-p 折中。
- **边界**：长文本生成需防重复（重复惩罚 repetition penalty）、防跑题（系统提示、RAG，见 [05/004 RAG](../05-大语言模型与Transformer/004-检索增强生成RAG.md)）。

## 七、一句话总结

- 文本生成 = 语言模型 + 自回归接龙；训练目标是最大化"下一个 token"的似然。
- 解码策略（贪心 / Beam / 采样）与温度、Top-k/Top-p 控制质量与多样性。
