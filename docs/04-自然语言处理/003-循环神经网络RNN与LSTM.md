# 003 · 循环神经网络 RNN 与 LSTM

> 本文用大白话回答：一句话有长有短，RNN 是怎么"边读边记"处理这种变长序列的？它为什么读到后面就忘了前面（记不住长距离信息）？LSTM 给记忆装上的"闸门"又是怎么把重要信息长期保住的？
>
> 读完你会知道：RNN 就是模仿人"从头读到尾、一边读一边记"；而 LSTM 是给这份记忆加了几道闸门，让该记的记牢、该忘的放掉。

## 一、一句话先说清

**RNN（循环神经网络）= 一个带"记忆"的读书机器：从左到右每读一个词，就用它更新一次记忆，再把记忆传给下一步。**

- 这份"记忆"叫**隐状态**（可以理解为"读到目前为止的内容摘要"）。
- 但普通 RNN 记性差，读到句尾常常忘了句首（**长距离依赖**问题）。
- **LSTM（长短期记忆网络）**给记忆加了几道**闸门**，专门解决"记不住长东西"的毛病。

## 二、打个比方：一边听故事一边记笔记

想象你在听一个很长的故事，手里拿着一张便签纸做笔记：

- **普通 RNN**：便签纸就那么大，每听一句你都要重写一遍笔记。写着写着，**开头的关键信息就被后面的内容一点点覆盖、稀释掉了**——听到结尾，早忘了主角是谁。
- **LSTM**：你多了一条"**重点档案袋**"（细胞状态），还有三个管理动作：
  - **遗忘门**：决定档案袋里哪些旧内容可以扔了；
  - **输入门**：决定这次听到的新内容哪些值得存进档案袋；
  - **输出门**：决定现在回答问题时，该从档案袋里取哪部分出来用。

档案袋的内容不会被反复重写覆盖，而是"该留的留、该换的换"，所以重要信息能**长期保留**。

| 生活里的角色 | RNN/LSTM 里的对应 | 干什么 |
| --- | --- | --- |
| 一边听一边写的便签 | 隐状态 | 保存"读到目前为止"的信息 |
| 长期不被随意覆盖的档案袋 | 细胞状态（LSTM） | 长期保留重要信息 |
| 扔旧内容 / 存新内容 / 取用内容 | 遗忘门 / 输入门 / 输出门 | 精细控制记忆的增删取用 |

## 三、它到底解决什么问题

### 问题 1：句子有长有短，怎么用一个模型都处理？——RNN 的参数共享

一句话可能 5 个词，也可能 50 个词。RNN 的巧妙之处是：**不管多长，都用同一套"读书 + 更新记忆"的规则，一个词一个词地滚下去。** 因为每一步用的是同一组参数，所以任意长度的序列都能处理。

### 问题 2：读到后面就忘了前面——长距离依赖与梯度消失

普通 RNN 有个根本毛病：**信息传得越远，衰减得越厉害。**

| 朴素做法（普通 RNN） | 更好做法（LSTM） |
| --- | --- |
| 记忆每步都被整体重写，早期信息层层稀释 | 重要信息放进"档案袋"，不被反复覆盖 |
| 训练时梯度沿长序列连乘，指数级衰减 | 记忆更新以"加法"为主，梯度能顺畅传播 |
| 学不到"隔很远的两个词之间的关系" | 能保住长距离依赖 |

这里的"梯度消失"可以这么直觉理解：训练时误差信号要**逐步往回传**，每回传一步就乘一次某个小于 1 的数，传过几十步之后就衰减成几乎 0 了——于是"很久以前那个词"对结果的影响学不出来（专业说法见第四节）。

### 问题 3：怎么让"该记的记住、该忘的忘掉"？——门控机制

普通 RNN 对记忆是"一刀切"地整体更新，没法区分"这条重要、那条没用"。LSTM 用三个**门**（可以理解为 0 到 1 的"阀门开合度"）来精细管理记忆：遗忘门管"扔什么"、输入门管"存什么"、输出门管"取什么"（专业公式见第四节）。

## 四、专业视角（与大白话对齐）

### 4.1 RNN 的循环结构

在时间步 $t$，用当前输入 $\mathbf{x}_t$ 与上一步隐状态 $\mathbf{h}_{t-1}$ 更新隐状态：

$$
\mathbf{h}_t = \tanh(W_x\mathbf{x}_t + W_h\mathbf{h}_{t-1} + \mathbf{b})
$$

同一组参数在所有时间步**共享**，因此能处理任意长度序列。

> 对齐：这个公式就是第二节说的"用上一步的记忆 $\mathbf{h}_{t-1}$ 加上这一步新读到的词 $\mathbf{x}_t$，写出新的记忆 $\mathbf{h}_t$"；参数共享正是第三节问题 1 里"不管多长都用同一套规则"。

### 4.2 梯度消失/爆炸

反向传播需沿时间展开（BPTT），梯度包含 $W_h$ 的连乘项。若其特征值 $<1$ 则梯度指数**消失**（学不到长依赖），$>1$ 则**爆炸**（训练发散）。这是普通 RNN 的根本缺陷。

> 对齐：这就是第三节问题 2 说的"每回传一步乘一次小于 1 的数，几十步后衰减成 0"的严谨版——特征值小于 1 时连乘导致指数消失。

### 4.3 LSTM 的门控机制

LSTM 引入细胞状态 $\mathbf{c}_t$ 与三个门（$\sigma$ 输出 0~1 的"开合程度"）：

$$
\begin{aligned}
f_t &= \sigma(W_f[\mathbf{h}_{t-1},\mathbf{x}_t]+b_f) &&\text{遗忘门}\\
i_t &= \sigma(W_i[\mathbf{h}_{t-1},\mathbf{x}_t]+b_i) &&\text{输入门}\\
o_t &= \sigma(W_o[\mathbf{h}_{t-1},\mathbf{x}_t]+b_o) &&\text{输出门}\\
\mathbf{c}_t &= f_t\odot\mathbf{c}_{t-1} + i_t\odot\tilde{\mathbf{c}}_t &&\text{更新记忆}\\
\mathbf{h}_t &= o_t\odot\tanh(\mathbf{c}_t)
\end{aligned}
$$

> 对齐：$f_t,i_t,o_t$ 就是第二节的"遗忘门 / 输入门 / 输出门"三个阀门；$\mathbf{c}_t$ 就是那条"重点档案袋"（细胞状态）；那句 $\mathbf{c}_t = f_t\odot\mathbf{c}_{t-1} + i_t\odot\tilde{\mathbf{c}}_t$ 正是"扔掉旧内容 $f_t\odot\mathbf{c}_{t-1}$、存入新内容 $i_t\odot\tilde{\mathbf{c}}_t$"。

关键在于 $\mathbf{c}_t$ 的更新是**加法**为主（而非反复矩阵相乘），梯度能较好地沿细胞状态传播，从而缓解梯度消失。GRU 是 LSTM 的简化版（合并门、更少参数）。

## 五、案例解析：为什么 LSTM 记得住"主语"

考虑句子："**那只**在院子里晒了一下午太阳、慵懒又肥硕的**猫**，忽然竖起耳朵，「**喵**」地叫了一声。"

要正确预测句尾的叫声是"喵"（而不是"汪"或"哞"），模型必须记住句子开头那个离得很远的主语是"猫"。普通 RNN 读到句尾时，早期的"猫"信息已在长序列中被稀释/遗忘，很可能预测出与主语不搭的词；而 LSTM 可通过遗忘门把细胞状态里"主语是猫"这一信息一路保留、不被中间那些描写冲掉，输出门在需要时再取用，从而正确处理这种**长距离依赖**。这直观解释了门控相较普通 RNN 的优势。

> 回扣：这正是第二节"档案袋"的价值——主语"猫"被存进细胞状态后一路不被覆盖，读到句尾要用时再由输出门取出。

## 六、常见误区与边界

- **误区："LSTM 彻底解决了长依赖"**：只是大幅缓解；超长序列仍力不从心，且**难以并行**（必须按时间步一个接一个地算）。
- **误区："RNN 已经过时无需学"**：RNN/LSTM 是理解序列建模与注意力动机的关键，且在小数据/流式场景仍有价值。
- **边界**：正是 RNN 的串行与长依赖瓶颈，催生了可并行、直连任意位置的 [注意力/Transformer](../05-大语言模型与Transformer/001-注意力机制与自注意力.md)。

## 七、一句话总结

- RNN 用一份共享参数的"循环记忆"（隐状态）从头读到尾处理变长序列，但记性差、有梯度消失和长依赖难题。
- LSTM 用遗忘 / 输入 / 输出三道门 + 一条"档案袋"（细胞状态），实现长期记忆；GRU 是它的简化版。
