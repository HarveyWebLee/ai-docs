# 004 · 文本分类与命名实体识别

> 本文回答：文本分类的完整流程是怎样的？命名实体识别（NER）为什么是"序列标注"？BIO 标注方案怎么用？

## 一、直觉与通俗解读

- **文本分类**：给**整段文本**贴一个标签。比如判断一条评论是"好评/差评"（情感分析），或一封邮件是"垃圾/正常"。相当于"读完整段，给个总评"。
- **命名实体识别（NER）**：在文本中**逐词**找出并标注人名、地名、机构名等实体。这不是给整句一个标签，而是给**每个词**一个标签，因此属于**序列标注**任务。相当于"边读边用记号笔圈出重点词"。

## 二、严谨定义与原理

### 2.1 文本分类流程

$$
\text{文本} \to \text{分词/子词} \to \text{向量表示} \to \text{编码器(RNN/CNN/Transformer)} \to \text{分类头(softmax)} \to \text{类别}
$$

- 损失用**交叉熵**（见 [01/004 信息论](../01-数学与理论基础/004-信息论基础.md)）。
- 二分类（好/差评）用 Sigmoid，多分类用 softmax。

### 2.2 序列标注与 BIO 方案

NER 给每个 token 一个标签。常用 **BIO** 标注：

- `B-XXX`：某类实体的开始（Begin）
- `I-XXX`：某类实体的内部（Inside）
- `O`：非实体（Outside）

例如对"马云 创办 了 阿里巴巴"：

| token | 马云 | 创办 | 了 | 阿里巴巴 |
| --- | --- | --- | --- | --- |
| 标签 | B-PER | O | O | B-ORG |

（PER=人名，ORG=机构名。多字实体会用 `B-` 接若干 `I-`。）

### 2.3 建模方法演进

- **传统**：CRF（条件随机场）建模标签之间的转移约束（如 `I-PER` 不能紧跟 `O`）。
- **深度学习**：BiLSTM 提取上下文特征 + CRF 解码（经典 BiLSTM-CRF）。
- **预训练时代**：BERT 等编码器 + 分类/标注头，效果大幅提升。

## 三、案例解析：情感分析的端到端流程

任务：判断影评"这部电影节奏拖沓，浪费了两小时"的情感极性。

1. **切分**：分词/子词得到 token 序列。
2. **表示**：每个 token 转为词向量（见 [001](./001-文本表示与词向量.md)）。
3. **编码**：用 BiLSTM 或 Transformer 编码整句，得到句子表示（如取 `[CLS]` 向量或做池化）。
4. **分类**：句向量过全连接 + softmax，输出 `[负面: 0.94, 正面: 0.06]`。
5. **判定**：负面概率高，判为**差评**。

训练时用大量已标注影评最小化交叉熵。这条"切分→表示→编码→分类"的流水线是几乎所有文本分类任务的通用范式。

## 四、常见误区与边界

- **误区："NER 是分类任务"**：NER 是**序列标注**，逐 token 输出，且标签间有依赖（适合 CRF 建模）。
- **误区："分类只看关键词"**：否定、反讽、上下文会翻转情感（"不好看"），需上下文建模。
- **边界**：标注数据成本高；少样本场景可借助预训练模型微调或提示（见 [09 提示工程](../09-提示工程与Agent开发/000-分类总览与知识图谱.md)）。

## 五、小结与延伸阅读

- 文本分类给整段贴标签（交叉熵）；NER 是逐 token 的序列标注，常用 BIO 方案。
- 通用范式：切分 → 表示 → 编码 → 任务头；CRF 擅长约束标签转移。
