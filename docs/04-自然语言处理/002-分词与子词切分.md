# 002 · 分词与子词切分

> 本文用大白话回答：模型读一句话之前，为什么非得先把它"切开"？中文没有空格，切错了会怎样？子词切分（BPE）是怎么用"拼积木"的思路，既控制住词表大小、又能拼出任何没见过的生僻词的？为什么今天的大模型几乎都用子词 token？
>
> 读完你会知道：切词不是小事——切法直接决定了模型看到的"最小单位"是什么，而子词切分是当前公认的最佳折中方案。

## 一、一句话先说清

**分词 = 在把文本喂给模型之前，先把它切成一个个"最小处理单位"（token，词元）。**

模型没法直接"吞"一整句话，得先切成一颗颗 token 才能逐个处理。而"该切多细、从哪切"是一门学问：切太粗，生僻词没法表示；切太细，又丢了词义。**子词切分**就是这两者之间的黄金折中。

## 二、打个比方：切菜与拼乐高

把"处理一句话"想成"做一道菜"：食材（整句话）不能整个下锅，得先切。

- 切太大块（**按整词切**）：遇到菜谱里没写过的新食材（生僻词、新词），厨师直接懵了——不认识，没法处理。
- 切成粉末（**按单字/字符切**）：什么都能处理，但一粒粒的粉末已经尝不出"这原本是什么菜"了（丢了词义）。
- **切成大小合适的块（子词切分）**：常见食材整块保留，罕见食材拆成认识的小块——既好处理，又保住了味道。

再换个角度看子词，它更像**拼乐高**：手里备好一批通用小积木（子词），常见的词直接用一整块，没见过的新词就用这些小积木现拼出来。这样积木种类（词表）不用太多，却能拼出无穷多的词。

## 三、它到底解决什么问题

### 问题 1：中文没有空格，从哪儿下刀？——分词的歧义

英文天然有空格帮忙断词，但中文是一长串连着的字，切错一个地方，**整句意思就变了**。

| 例子 | 一种切法 | 另一种切法 | 后果 |
| --- | --- | --- | --- |
| 研究生命科学 | 研究 / 生命 / 科学 | 研究生 / 命 / 科学 | 切错语义全变 |

而且英文也不是完全没烦恼：`playing = play + ing`，还有各种罕见词、拼写变体，整词处理起来同样别扭。

### 问题 2：新词永远层出不穷，词表怎么装得下？——未登录词（OOV）

无论词表事先收录多少词，现实里总会冒出没见过的：新造的网络词、人名、专有名词、商品名……这些不在词表里的词就叫**未登录词（OOV，Out-Of-Vocabulary）**。

| 朴素做法（按整词切 + 固定词表） | 更好做法（子词切分） |
| --- | --- |
| 词表必须尽量收全，越收越大 | 只需几万个"零件"，词表大小可控 |
| 一遇到没收录的新词就抓瞎（OOV） | 任何新词都能用零件拼出来，最差退回到字符/字节 |
| "slowest"没收录 → 无法表示 | "slowest" → 拼成 `slow` + `est` |

**子词切分**正是折中方案：把词拆成更小的"零件"，**常见词整体保留，生僻词拆成子词**。这样既控制了词表大小，又能拼出任何没见过的词。

### 问题 3：怎么自动决定"哪些该整块保留、哪些该拆"？——BPE 的合并思路

我们不想手工规定规则，而是让算法**从语料里自己统计**：谁总是黏在一起出现，就把谁合并成一块。这就是 BPE（详见第四节的严谨步骤与第五节的演示）。

## 四、专业视角（与大白话对齐）

### 4.1 中文分词的挑战

- **歧义切分**："乒乓球拍卖完了"可切成"乒乓球拍/卖完了"或"乒乓球/拍卖/完了"。
- **未登录词（OOV）**：新词、人名、专有名词层出不穷。

经典方法：基于词典的最大匹配、基于统计的隐马尔可夫模型（HMM）/条件随机场（CRF）、以及基于神经网络的序列标注。

> 对齐：这正是第三节问题 1、问题 2 说的"从哪下刀会有歧义"和"新词装不进词表"。

### 4.2 子词切分：BPE

**字节对编码（Byte Pair Encoding, BPE）** 的核心思想：

1. 初始把文本拆成最小单元（字符/字节）。
2. 统计相邻单元对的出现频率，把**最高频的对**合并成一个新单元。
3. 重复合并直到达到预设词表大小。

> 对齐：这就是第二节"拼乐高"的正式版——**谁总黏在一起（高频相邻），就把谁合并成一块新积木**。

这样高频组合（如 `ing`、`tion`）被合并为整体，低频词自然被拆成子词。相关变体有 WordPiece（BERT）、Unigram/SentencePiece（多语言、无需预分词）。

### 4.3 为什么大模型用子词

- **词表可控**：几万个子词即可覆盖任意语言的任意词。
- **无 OOV**：任何词都能由子词拼出，最差回退到字符/字节。
- **跨语言**：字节级 BPE 可统一处理多语言与表情符号。

> 对齐：这三点正是第三节问题 2 表格里"词表可控""任何新词都拼得出""退回字符/字节兜底"的专业说法。

## 五、案例解析：BPE 合并过程演示

假设语料只有单词 `low, lower, newest, widest`，初始按字符切分。统计相邻对频率，`e` 与 `s` 常相邻（来自 `newest, widest`），于是首先合并：

1. 合并 `e s → es`：`new es t`, `wid es t`
2. 合并 `es t → est`：`new est`, `wid est`
3. 合并 `l o → lo`、`lo w → low` …

于是 `newest` 被切为 `new` + `est`，`widest` 被切为 `wid` + `est`——**`est` 这个高频后缀被复用**。当遇到训练时没见过的新词 `slowest`，也能被切成 `slow` + `est`，从而不再是"未登录词"。这正是子词切分兼顾词表大小与泛化能力的关键。

> 回扣：这就是第二节说的"用备好的小积木现拼新词"——`est` 这块积木一旦学会，新词 `slowest` 直接借用即可。

## 六、常见误区与边界

- **误区："token = 词"**：一个 token 可能是子词、单字符甚至一个字节；模型说的"上下文长度"是按 token 计的，不是按词计的。
- **误区："中文一定按字切最好"**：按字切会丢失词义信息（"生命"拆成"生""命"就变味了），实际多用子词/词混合策略。
- **边界**：切分粒度影响序列长度与计算成本——切得越碎，token 越多、算得越慢，需在语义完整性与效率间权衡。

## 七、一句话总结

- 分词把文本切成 token，是喂给模型前的第一步；中文因为没空格，面临歧义切分与未登录词两大挑战。
- BPE 等子词方法用"高频就合并"的思路，既把词表大小控制住，又能拼出任意新词（消除 OOV），是现代大模型的标准做法。
