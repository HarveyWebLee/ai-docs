# 004 · 过拟合与正则化

> 本文回答：什么是过拟合/欠拟合？偏差-方差权衡说的是什么？L1/L2 正则化、Dropout、早停如何缓解过拟合？

## 一、直觉与通俗解读

- **过拟合**：像"**死记硬背**"的学生，把训练题的答案（甚至噪声）都背下来了，一考新题就露馅——训练集表现极好、测试集很差。
- **欠拟合**：像"**没学明白**"的学生，连训练题都做不好——模型太简单，训练集测试集都差。
- **正则化**：给模型"**立规矩**"，不让它把参数学得太极端、太复杂，从而更关注普适规律而非死记细节。

理想状态是**恰好拟合**：既学到规律，又不被噪声带偏。

## 二、严谨定义与原理

### 2.1 偏差-方差权衡

泛化误差可近似分解为：

$$
\text{期望误差} \approx \text{偏差}^2 + \text{方差} + \text{不可约噪声}
$$

- **偏差（Bias）**：模型假设过于简单导致的系统性错误（→ 欠拟合）。
- **方差（Variance）**：模型对训练数据波动过于敏感（→ 过拟合）。

模型复杂度增加时，偏差下降但方差上升；目标是找到**总误差最小**的平衡点。

### 2.2 L2 与 L1 正则化

在损失中加入参数惩罚项：

$$
J_{\text{reg}} = J(\theta) + \lambda\,\Omega(\theta)
$$

- **L2 正则（权重衰减）**：$\Omega=\sum_i w_i^2$，让权重整体变小、更平滑，缓解过拟合。
- **L1 正则**：$\Omega=\sum_i |w_i|$，倾向把部分权重压成 0，产生**稀疏解**，可用于特征选择。
- $\lambda$ 控制正则强度：过大导致欠拟合，过小则正则无效。

### 2.3 其它常用手段

- **Dropout**：训练时随机"关闭"一部分神经元，迫使网络不依赖单一路径（深度学习常用）。
- **早停（Early Stopping）**：监控验证集误差，一旦不再下降就停止训练。
- **数据增强 / 增加数据**：更多样的数据是对抗过拟合最根本的办法。

## 三、案例解析：多项式拟合的复杂度选择

用多项式拟合一批带噪声的散点：

- **1 次多项式（直线）**：太简单，拟合不了数据的弯曲趋势 → **欠拟合**（高偏差）。
- **15 次多项式**：曲线扭曲着穿过每一个训练点，把噪声也拟合了 → **过拟合**（高方差），新点预测极差。
- **3~4 次多项式**：既抓住主趋势又不追噪声 → **恰好拟合**。

若坚持用高次多项式，可加 L2 正则**惩罚过大的系数**，把扭曲的曲线"拉平滑"，从而在高复杂度下依然获得良好泛化。这直观展示了"复杂度 + 正则化"共同决定泛化的机制。

## 四、常见误区与边界

- **误区："训练误差越低越好"**：训练误差过低而验证误差高，正是过拟合信号。
- **误区："正则化越强越好"**：过强正则会欠拟合，$\lambda$ 需用验证集调优。
- **误区："过拟合只靠正则化解决"**：更多/更干净的数据、更合理的模型复杂度往往更有效。

## 五、小结与延伸阅读

- 过拟合=记噪声（高方差），欠拟合=没学会（高偏差），核心是偏差-方差权衡。
- L2 平滑权重、L1 产生稀疏、Dropout/早停/数据增强各有用武之地。
